## 一、产品目标与角色

### 1.1 目标
- 承接 Leapexbiz 小程序产生的全部业务：客户、KYC、核名、订单、账单、交付物、年审、变更。
- 满足香港 AMLO / 公司条例 / TCSP 牌照的合规运营与记录保存（7 年、不可篡改）。
- 支撑多渠道分销：渠道差异化定价 + 渠道佣金 + 供应商协同交付，财务可对账。

## 三、核心运营模块（与小程序闭环对齐）

> **闭环原则**：小程序每一个"提交/等待"动作，后台都有对应的"审核/处理"入口；处理结果通过通知回推小程序。

| 小程序客户端动作 | 后台对应模块 | 处理结果回推 |
|---|---|---|
| 提交 KYC（5 步） | KYC 审核（+EDD/制裁筛查） | 通过/补充/拒绝 → 通知 + 状态 |
| 提交核名 | 核名查册（iCRIS + 风险评级） | 风险等级 + 锁定 30 天 → 通知 |
| 上传汇款回单 | 回单审核（财务确认） | 已到账/驳回 → 订单进入服务中 |
| 等待交付物 | 交付物管理（PDF 生成 + 人工/供应商上传） | 交付物就绪 → 交付物中心可见 |
| 年审到期 | 年审任务 + 6 轮递进提醒 | 提醒触达 + 生成年审账单 |
| 问顾问 / 预约咨询 | 企微线索 / 工单 | 顾问跟进 |

### 3.1 客户管理
列表（按名称/邮箱/电话搜索、按渠道/状态筛选）· 客户详情（KYC 资料、订单历史、公司列表、企微线索）· KYC 资料查看/打包下载 · 内部备注 · 标签（高价值/需跟进）· **被拒客户 7 年留存区**。

### 3.2 KYC / EDD 审核
- 队列：待审核（含提交时间、SLA 倒计时）。
- 审核台：证件 OCR 结果 + 原件、地址证明、业务声明、PEP 自评、**制裁筛查结果**。
- 动作：通过 / 要求补充（退回小程序）/ 拒绝（留档 7 年，记拒绝原因）/ 标记 EDD。
- **EDD 流程**：PEP/高风险 → 要求资金来源证明 + 业务关系合理性 → 合规二次审核。
- **制裁命中**：自动冻结订单 → 合规复核 → 解冻 / 上报（STR 预留）。
- KYC 状态机：`draft → submitted → reviewing → approved | edd_required | rejected`；命中制裁 → `frozen`。

### 3.3 核名查册
队列（智能预检通过待查册）· iCRIS 查册结果上传（截图存档）· 五级风险评级（Green/Yellow/Orange/Red/Blue）· 名称锁定 30 天台账 · 免费重查 3 次计数（可手动增加，超额计费）· 历史核名记录查询。

### 3.4 订单管理
列表（订单号/客户/服务/状态/渠道筛选）· 订单详情（全流程 KYC→核名→签约→注册→交付时间轴）· 操作日志（不可篡改）· 内部备注 · 导出 Excel（对账）。
> 订单 > 服务 > 账单三级；一个订单可含多服务，每服务 1:1 一张账单。

### 3.5 账单 / 支付管理（财务核心）
- 账单列表（状态/时间/金额/渠道）· 回单审核（看截图，确认/驳回）· 手动生成账单 · **账单作废 / 重开（≤2 次/订单）/ 退款 → Finance 审批 + 留痕**。
- 账单状态机：`待支付 → 待确认 → 已到账 → 服务中`；旁支 `已作废（7天未付）/ 已驳回`。
- 首付款(50%) + 尾款(50%) 双账单；**1:1 支付映射**（payment.bill_id UNIQUE）。
- **收入确认时点**：账单"已到账"即确认收款（现金制记录），对账以银行实际入账为准；导出供做账/审计。
- 印花税：股份转让账单自动按 **从价 0.2%（对价与资产净值孰高）** 预估并在账单注明。

### 3.6 交付物管理
模板管理（HTML+CSS，`{{变量}}` 占位，Puppeteer 渲染）· PDF 生成测试 · 人工/供应商上传扫描件 · 快递单号录入（客户端可追踪）· 版本管理。
> 新注册 19 项（客户端可见 17）；其他 8 服务 67 项。KYC 存档(D3) 内部不可见客户端。

### 3.7 年审与变更任务
年审台账（到期日、6 轮递进提醒触发）· 到期前 60 天自动生成年审账单 · 变更类服务（董事/秘书/地址/股份/名称/配发/注销）工单与交付物流转 · **SCR 联动更新**。

### 3.8 企微承接 / 线索工单（新增）
- 小程序「问顾问 / 预约咨询」→ 线索池（含来源埋点：服务详情/下单页/悬浮/合作方服务）。
- 字段：客户、来源、意向服务、状态（待跟进/跟进中/已转化/关闭）、负责顾问、跟进记录。
- 价值：定位最佳承接时点（呼应商业策划的"承接缺口"），沉淀转化数据。

## 四、定价与渠道佣金（财务双台账）

### 4.1 客户侧定价矩阵（渠道 × 服务）
每格独立配置固定价/折扣率，留空继承标准价，支持生效/失效日期；小程序按 Channel Code 展示渠道专属价；账单按渠道专属价计算；自然流量按标准价。

### 4.2 渠道佣金（独立于定价）
- 佣金率矩阵（渠道 × 服务），留空不计佣。
- **佣金基数 = 客户实付（渠道专属价）× 佣金率**，订单"已到账"时生成应付佣金记录。
- 月度佣金汇总 → 渠道应付台账 → 线下转账 → 标记"已结算"。
- 渠道 Admin 仅见本渠道佣金。

### 4.3 渠道应付台账（新增，财务）
| 字段 | 说明 |
|---|---|
| 佣金记录 ID | 单笔佣金 |
| 渠道 / 订单 / 服务 | 关联 |
| 基数（实付）/ 佣金率 / 佣金额 | 计算留痕 |
| 周期 / 状态（待结算/已结算）/ 结算时间 | 月结对账 |

## 五、供应商模块（应付分账）

### 5.1 供应商管理
账号创建（Admin 分配）· 档案（公司/联系人/邮箱/服务类型）· 状态（启用/暂停）· 服务范围（负责的交付物类型）。

### 5.2 交付任务
派发流程：运营建任务 → 指派供应商 → 供应商接收 → 上传文件 → 运营确认/驳回。
任务状态：`待接收 → 进行中 → 已上传 → 已确认 | 已驳回`。

### 5.3 供应商应付账单（线下结算）
管理员手动创建（供应商/服务内容/金额/周期/关联订单可多对一）· 结算状态（待结算/已结算）· 供应商只读自己账单 · Finance 只读 + 标记结算。
> 与渠道佣金应付构成**两本应付台账**，月度统一对账。

## 六、合规与系统

### 6.1 合规管理
制裁名单管理（UN/EU/HKMA 更新）· KYC 记录导出（含被拒，供审计）· 合规报告（按周期生成 AMLO 要求报告）· **可疑交易报告 STR 预留** · 操作日志审计（不可篡改）· **SCR 登记册维护**。

### 6.2 通知管理
模板（订阅消息/站内信/邮件）· 发送记录 · 年审递进规则可配置（轮次/时机/模板）· 邮件服务商配置（SendGrid/SES/阿里云）· **合规通知不可退订**（KYC 结果/核名/年审逾期/制裁/EDD）。

### 6.3 系统配置（参数）
`payment_deadline_days=7` · `name_lock_days=30` · `name_check_free_quota=3` · `kyc_audit_timeout_hours=48` · `data_retention_years=7` · `bill_prefix=LEA-` · `order_prefix=ORD-` · `supplier_bill_prefix=SUP-` · `company_kit_assembly_days=3` · `seal_making_days=2` · `annual_review_advance_days=60` · `max_bill_reopen_count=2` · `stamp_duty_rate=0.002`。

### 6.4 审计与数据安全
AES-256 字段级加密（身份证号/地址/UBO）· 全站 HTTPS/TLS 1.3 · RBAC · **审计日志哈希链不可篡改** · 每日增量+每周全量备份 · KYC（含被拒）7 年留存 · PDPO 客户查阅/更正权。
