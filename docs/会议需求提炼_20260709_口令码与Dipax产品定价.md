## 一、渠道标识：渠道码 → 口令码 / 区域码（★ P0 · 核心决策）🔧改造

**问题**：现"渠道码"会让客户察觉自己是通过**中介**对接，即便价格一致也抵触"被赚差价"；且无码的自然流量会造成**渠道归属纠纷**。

- **改名**：`渠道码` → **口令码 / 区域码**（从措辞上淡化"渠道 / 中介"属性）。小程序 + 后台**全量替换文案**。
- **包装成福利**：对外统一解释为「**销售业绩核算专属标识**」，并**绑定该销售的专属优惠券权益** → 把生硬的渠道标识变成"**领优惠券的入口**"，降低一线销售解释成本。
- **强制输码（★）**：**彻底取消自然流量入口**，所有用户**无论来源都必须输入对应口令码**才能进入服务流程；直客输入自己对应的码即可。
  - 现状 ✅：上一版已把首屏改为"渠道码必填 + 去掉跳过"（[[会议需求提炼_20260630_注册交付与渠道]] 已提强制）。本场进一步：**措辞改口令码 + 明确"绝无自然流量"**。
- **落地动作**：① 首屏与首页 hero 文案 `渠道码` → `口令码`；② 口令码页副本改为"领取专属优惠券的口令"；③ 后台渠道商模块 `渠道码 / channel code` → `口令码 / 区域码`（保留 code 字段，仅改展示名）。
