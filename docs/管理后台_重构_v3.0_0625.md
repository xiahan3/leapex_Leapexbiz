## 〇、v3.0 对 v2.0 的纠正（务必落实）

| # | v2.0 现状 | v3.0 纠正 | 原因 |
|---|---|---|---|
| C1 | 只有「渠道商管理」一个模块（供应商并入订单组） | **拆成 渠道商管理 + 供应商管理 两个独立模块** | 渠道商=投流获客、收佣金；供应商=履约乙方、按金额/比例结算，二者本质不同 |
| C2 | 「定价矩阵」= 渠道 × 服务，每渠道一个**售价** | **删除定价矩阵**；价格=商品**标准价**（商品管理维护），渠道差异只体现在**佣金** | 售价跟服务走，不是每渠道一价 |
| C3 | 商品分"套餐 / 单项"，套餐另挂字段 | 商品**字段统一**：单项 / 组合服务同一 schema | 否则无法整合、无法统一推荐 |
| C4 | 渠道商工作台 1 个 | **两个工作台**：供应商工作台 + 渠道商工作台 | 履约方与获客方各自登录、各看各的 |

## 一、模块结构（8 大模块）

| # | 模块 | 内含 | 对应小程序 |
|---|---|---|---|
| 1 | **数据驾驶舱** | KPI + 待办 + 业务流程总览 | 全局 |
| 2 | **客户管理** | 客户信息 · 客户公司 · **核名审核** · **KYC 审核** | 推荐码 / 登录 / 核名 / 我的公司 |
| 3 | **订单管理** | 订单 · 付款 · 交付物 ·（**不含供应商**，分配是独立动作） | 下单 / 查看订单 / 订单完结 |
| 4 | **商品管理** | 单项 / 组合服务，**统一字段**，上下架，**标准价**（含双币） | 服务市场 / 推荐 |
| 5 | **供应商管理** | 履约乙方：入驻 → **结算规则**（固定额/比例）→ 订单分配 → 进度 → 结算 → 提现 | 履约方 |
| 6 | **渠道商管理** | 投流获客：channel code → 按**服务/订单**配**佣金（返点）** | 渠道码 |
| 7 | **工作台（×2）** | **供应商工作台** + **渠道商工作台**：我的订单 / 维护进度 / 结算 / 提现 | 乙方/渠道侧 |
| 8 | **线索与投流** | **线索登记/跟进（一期）** + **投流数据（二期）** | 免费咨询 / 获客承接 |

> 用户及权限（RBAC）、合规审计、系统配置作为支撑模块并入相应位置；生态合作伙伴（办公租赁）按"供应商/服务方"纳入（订单分配 + 工作台），命名"供应商 or 生态"待业务定。

## 二、业务流程（驾驶舱已落图）

**客户**：填推荐码 → 登录 → 核名 → 看服务 → 购买 → 查看订单 → 订单完结
**供应商**：入驻 + 结算规则 → 订单分配 → 维护进度 → 看结算 → 提现
**渠道商**：入驻 + 佣金规则 → 带来订单(channel code) → 看佣金 → 结算提现

## 三、关键规则

### 3.1 渠道商 vs 供应商（C1，核心区分）

| 维度 | 渠道商 | 供应商 |
|---|---|---|
| 角色 | 投流 / 推广 / 获客（前端 channel code 来源） | 履约 / 提供服务（乙方） |
| 钱的方向 | 客户付给平台，平台给渠道商**返点/佣金** | 平台收钱后，按**固定额 / 比例**结算给供应商 |
| 配置 | 渠道 × 服务 **佣金率**（可逐服务/订单不同） | 服务 × 供应商 **结算金额/比例**（可逐供应商不同） |
| 工作台 | 渠道商工作台（看带来的订单 + 佣金 + 提现） | 供应商工作台（看分到的订单 + 进度 + 结算 + 提现） |
| 生态合作（办公租赁） | — | 归此类（他收钱分平台，仍需订单+工作台跟进） |

### 3.2 价格与结算（C2）
- **售价** = 商品标准价（商品管理维护，双币）；**无"每渠道一价"**，删除定价矩阵。
- **渠道佣金** = 客户实付 × 渠道×服务佣金率（渠道商管理配）。
- **供应商结算** = 固定金额 or 实付×比例（供应商管理配）。
- **结算时点**（可配）：月结 / 每单结 / **用户确认收货后结**（类淘宝，N 天自动确认）。

### 3.3 商品字段统一（C3）
单项 / 组合服务同一 schema：类型、单价（双币）、包含明细、服务内容、**适用人群**、交付门槛、**前置流程**（是否需核名 / KYC / 前置资料）。前置流程字段驱动小程序办理步骤。

### 3.4 订单分配
无推荐码（自然流量/投流）订单 → 运营按规则（地域/负载/优先级，兜底转直营）分配给**供应商**履约；带 channel code 的订单自动归属对应**渠道商**计佣。
