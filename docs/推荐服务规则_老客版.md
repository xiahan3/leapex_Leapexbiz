# Leapexbiz 老客版「推荐服务」规则 v1.0

> **用途**：老客（已注册+已付款）首页「为你推荐」模块的服务推荐逻辑。
> **目标**：用企业生命周期 + 合规节点 + 用户画像，推出"此刻最该办"的服务，驱动复购，避免逾期罚款。
> **原则**：合规刚需优先 · 临期置顶 · 已购去重 · 理由可解释。

---

## 一、推荐输入（信号源）

| 维度 | 信号 | 来源 |
|---|---|---|
| 企业状态 | 注册中 / 正常运营 / 年审待办 / 年审逾期 / 变更中 / 注销中 | company.status |
| 合规节点 | 年审到期日、财政年度结算日、BR 到期、SCR 更新需要 | company 字段 |
| 历史订单 | 已购/进行中的服务（用于去重）、上次办理时间 | order/bill |
| 用户画像 | 多公司客户、出海/跨境业务、是否需身份、客单价层级 | customer.tags / source |
| 渠道 | 渠道客户（专属价/专属推荐位） | channel_code |

---

## 二、推荐规则（按优先级，从高到低）

### P0 · 合规刚需（强制/有法定期限，置顶、红/橙标）
| 触发条件 | 推荐服务 | 理由文案 |
|---|---|---|
| 年审到期 ≤60 天 | **公司年审 NAR1** | 「距到期 N 天 · 逾期罚款递增」 |
| 年审已逾期 | **紧急年审 + 罚款处理** | 「已逾期 N 天 · 影响公司合规状态」（最高优先级，红色）|
| 财政年度结算日已过 | **做账 + 审计 + 利得税申报** | 「财年已结束 · 须按时报税」 |
| BR 商业登记证将到期 | **BR 续期** | 「BR 到期前续办」 |
| 董事/股东/股份发生变更迹象 | **对应变更 + SCR 更新** | 「保持 SCR 与名册同步」 |

### P1 · 业务衔接（生命周期顺延，自然下一步）
| 触发条件 | 推荐服务 | 理由文案 |
|---|---|---|
| 注册即将/刚完成 | **银行开户协助** | 「注册完成即可启动开户」 |
| 新公司无记账安排 | **月度记账报税** | 「新公司建账，按月合规」 |
| 有雇员/将招人 | **MPF 强积金 / 雇员保险** | 「雇主须为雇员供 MPF」 |
| 注册地址到期/借用他址 | **注册地址续约 / 迁址** | 「保持注册地址合规」 |

### P2 · 增值/画像（按客户画像，软推荐）
| 触发条件 | 推荐服务 | 理由文案 |
|---|---|---|
| 已设立香港公司、出海创业者画像 | **香港移民 / 身份规划（优才/高才）** | 「已设立公司，可规划身份」 |
| 多公司客户 | **批量年审 / 集团记账打包** | 「多家公司统一托管更省」 |
| 高客单画像 | **1 对 1 顾问 / 公证认证** | 「专属顾问深度服务」 |

---

## 三、排序与展示规则

1. **优先级排序**：P0 > P1 > P2；同级内按"临期紧迫度"（到期天数越小越靠前）。
2. **逾期/紧急置顶**：任何"已逾期/罚款风险"项强制置顶并用红色标。
3. **每公司聚合**：推荐卡标注归属公司（如「恒丰贸易 · 临期」），客户一眼知道是哪家。
4. **去重与冷却**：
   - 已购买/进行中的服务**不再推荐**（查 order/bill）。
   - 被用户忽略/关闭的推荐，进入**冷却期**（如 14 天）不再出现。
5. **数量上限**：首页"为你推荐"最多展示 **4 条**，其余进"全部服务"。
6. **可解释**：每条都带"为什么推荐你"的理由文案（合规/节点/画像），不做黑盒。

## 四、计算伪代码

```text
recs = []
for company in user.companies:
    # P0 合规刚需
    days = company.next_ar_date - today
    if company.status == 'ar_overdue':
        recs.add(svc='年审(紧急)', company, priority=0, urgent=True, reason='已逾期 %d 天')
    elif 0 <= days <= 60:
        recs.add(svc='年审 NAR1', company, priority=0, reason='距到期 %d 天' % days)
    if past(company.fy_end):
        recs.add(svc='做账+报税', company, priority=0, reason='财年已结束')
    # P1 业务衔接
    if company.status in ('registering','active') and not has_service(company,'bank_open'):
        recs.add(svc='银行开户协助', company, priority=1, reason='注册完成即可开户')
    if not has_service(company,'bookkeeping'):
        recs.add(svc='记账报税', company, priority=1, reason='按月合规建账')
# P2 画像
if user.has_tag('出海') and user.companies:
    recs.add(svc='香港移民/身份', priority=2, reason='已设立公司可规划身份')
if len(user.companies) >= 2:
    recs.add(svc='集团记账/批量年审', priority=2, reason='多公司统一更省')

recs = dedup(recs, exclude=purchased_or_inprogress)      # 去重
recs = filter(recs, not_in_cooldown)                     # 冷却
recs = sort(recs, by=[priority asc, urgent desc, days asc])
return recs[:4]                                           # 首页最多 4 条
```

## 五、埋点

| 触发时机 | 业务意义 |
|---|---|
| 推荐曝光（按 service×company） | 推荐覆盖与命中分布 |
| 点击推荐卡 | 推荐点击率（CTR） |
| 推荐→下单→付款 | 推荐转化率（核心复购指标） |
| 忽略/关闭推荐 | 负反馈，进冷却 |

## 六、原型现状（演示用静态推荐）

老客版首页「为你推荐」当前演示 4 条（按上述规则人工排好）：
1. 公司年审 NAR1（恒丰贸易·临期，P0）
2. 记账报税 + 利得税（恒丰贸易·年度，P0/P1）
3. 银行开户协助（星辰科技·注册后，P1）
4. 香港移民/身份规划（出海画像，P2）
