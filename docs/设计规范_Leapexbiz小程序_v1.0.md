# EasybookX 公司秘书小程序 — 设计交互规范 v1.0

> **适用范围**：EasybookX TCSP 公司秘书微信小程序（服务目录 / 电商购买 / KYC 尽调 / 核名注册 / 账单支付 / 订单跟踪）。
> **设计基因**：以 **Claude 移动端设计语言**（陶土橘 + 克制中性灰 + 暗色适配）为视觉底座，融合**微信小程序平台规范**（rpx、胶囊避让、TabBar、订阅消息、授权时机、WeUI 单元格）。
> **与 Claude 对话规范的差异**：本应用非对话式 AI，**剔除**流式输出 / 消息气泡 / 思考指示等聊天组件，**保留**其设计基础（Tokens / 排印 / 间距 / 暗色 / 无障碍 / 动效 / 反馈），并新增**服务电商 + 表单 + 合规**场景组件。
> **更新**：2026-06-11

---

## 1. 设计原则

| 原则 | 含义 | 落地要求 |
|------|------|----------|
| 服务优先 | 界面为「办成一件事」服务，而非堆功能 | 每屏聚焦单一任务（选服务 / 填信息 / 付款），主操作唯一 |
| 即时可感 | 用户随时知道系统在做什么 | 解析、查册、审核、支付确认都有明确状态反馈 |
| 克制视觉 | 内容是主角，容器是配角 | 少用强色块/重阴影，用留白与 1px 细线分隔；强调色（陶土橘）谨慎使用 |
| 可中断可回退 | 长流程能随时退出、续办 | 多步流程顶部步骤条 + 可返回；草稿自动保存 |
| 单手友好 | 高频操作落在拇指热区 | 主操作（立即办理 / 提交 / 支付）居底部固定栏；危险操作远离误触区 |
| 合规可信 | TCSP 持牌、AMLO 合规外显 | KYC / 资料保存 / 渠道归属等合规信息清晰可见，建立信任 |

## 2. 设计基础（Design Tokens）

### 2.1 色彩系统（语义化，支持深浅色）

采用语义 Token，禁止页面里直接写色值；通过 WXSS 变量切换深浅色。

#### 基础中性 + 品牌

| 语义 Token | 浅色 | 深色 | 用途 |
|-----------|------|------|------|
| `bg/canvas` | #FFFFFF | #1A1A1A | 页面底色（OLED 深色不用纯黑，减拖影） |
| `bg/surface` | #F5F5F2 | #262626 | 卡片 / 输入栏 / 次级背景 |
| `bg/sunken` | #EFEFEC | #121212 | 列表分组背景 / 凹陷区 |
| `text/primary` | #1A1A1A | #F5F5F2 | 正文 / 标题 |
| `text/secondary` | #6B6B6B | #A0A0A0 | 辅助 / 时间 / 说明 |
| `text/tertiary` | #9B9B9B | #6E6E6E | 占位符 / 禁用 |
| `accent/primary` | **#C15F3C** | **#D97757** | 主操作、链接、强调（陶土橘 · Claude 品牌） |
| `accent/weak` | #F7EBE5 | #3A2A22 | 强调底色（选中态 / 标签底） |
| `border/divider` | #E5E5E2 | #3A3A3A | 分隔线 / 卡片描边（1px） |

#### 语义状态（服务 / 订单 / 账单）

| 语义 Token | 浅色 | 深色 | 用途 |
|-----------|------|------|------|
| `state/success` | #2E8B57 | #4CAF7D | 成功 / 已到账 / 已完成 |
| `state/warning` | #B45309 | #E0A458 | 待支付 / 待办 / 风险提示 |
| `state/info` | #3B6EA5 | #6F9BD1 | 服务中 / 处理中 |
| `state/error` | #D93A3A | #F06A6A | 失败 / 驳回 / 校验错误 |
| `state/neutral` | #8A8A8A | #7A7A7A | 已取消 / 已作废 / 已含 |

> 状态色仅用于「状态指示」，不做大面积铺色。每个状态标签同时用**文案 + 弱底色**，不以颜色为唯一信息载体。

### 2.2 字体排印（小程序 rpx）

> 小程序设计基准宽度 **750rpx**（≈ 375pt 屏）。换算：**1pt ≈ 2rpx**。下表给出 rpx 与等效 sp。

| 层级 | 字号 (rpx / sp) | 行高 (rpx) | 字重 | 用途 |
|------|----------------|-----------|------|------|
| Display | 44 / 22 | 60 | 700 | 启动页大标题 |
| Title | 36 / 18 | 52 | 600 | 页面 / 卡片标题 |
| Body-L | 32 / 16 | 48 | 400 | 正文（关键值，可点字号 ≥32rpx） |
| Body | 28 / 14 | 44 | 400 | 常规列表 / 表单值 |
| Caption | 24 / 12 | 34 | 400 | 时间 / 状态 / 辅助说明 |
| Mono | 28 / 14 | 40 | Mono | 账号 / 账单号 / 金额 |

- 正文最小可点字号 **≥ 28rpx**，避免缩放。
- 价格金额统一用 `text/primary` 或 `state/error`（促销价）+ **700 字重**，币种「HKD」小一号前缀。
- 支持系统「字体大小」无障碍设置（小程序 `rich-text` / 跟随系统），布局随之伸缩。

### 2.3 间距与栅格

- 基础栅格 **8rpx**；常用：8 / 16 / 24 / 32 / 48 / 64 rpx。
- 屏幕安全边距：左右 **32rpx**。
- 卡片内边距：**28rpx**。
- 列表项（Cell）高度：**≥ 96rpx**；行内边距 28rpx。
- 区块间距：同组 16rpx，不同组 24–32rpx。

### 2.4 圆角 / 阴影 / 图标

- 圆角：卡片 / 输入栏 **16rpx**，按钮 **16rpx**，弹层 **24rpx**，小标签 **12rpx**，头像 / 图标块 **18rpx**。
- 阴影：移动端**优先用 `border/divider` + 底色差异**表达层级；仅浮层 / 底部购买栏用轻阴影 `0 -2rpx 16rpx rgba(0,0,0,.06)`。
- 图标：**线性图标，2px(4rpx) 描边，24×24pt 标准**；点击热区 **≥ 88rpx (44pt)**。
  - 原型用 Tabler 线性图标；上真机改为 **iconfont / 本地 SVG symbol**（小程序不支持 webfont CDN，需内置）。
  - 全局同一语义同一图标（见 §附录 A 图标映射）。

### 2.5 WXSS 变量落地示例

```css
/* app.wxss · 浅色（:root 概念，小程序用 page 或全局类） */
page {
  --bg-canvas:#FFFFFF; --bg-surface:#F5F5F2; --bg-sunken:#EFEFEC;
  --text-1:#1A1A1A; --text-2:#6B6B6B; --text-3:#9B9B9B;
  --accent:#C15F3C; --accent-weak:#F7EBE5;
  --line:#E5E5E2;
  --success:#2E8B57; --warning:#B45309; --info:#3B6EA5; --error:#D93A3A; --neutral:#8A8A8A;
  --radius-card:16rpx; --radius-btn:16rpx; --radius-tag:12rpx;
  --safe-bottom: env(safe-area-inset-bottom);
}
/* 深色：跟随系统 */
@media (prefers-color-scheme: dark){
  page{
    --bg-canvas:#1A1A1A; --bg-surface:#262626; --bg-sunken:#121212;
    --text-1:#F5F5F2; --text-2:#A0A0A0; --text-3:#6E6E6E;
    --accent:#D97757; --accent-weak:#3A2A22; --line:#3A3A3A;
    --success:#4CAF7D; --warning:#E0A458; --info:#6F9BD1; --error:#F06A6A; --neutral:#7A7A7A;
  }
}
```
> 小程序也可用 `wx.onThemeChange` + `theme` 字段做主题切换；`app.json` 设 `"darkmode": true` 并配 `themeLocation`。

## 3. 布局与导航

### 3.1 整体框架

```
┌─────────────────────────┐
│  状态栏(系统)            │  ← safe-area-inset-top
│  导航栏 + 右上胶囊       │  ← 自定义导航须避让胶囊(约 87pt 宽)
├─────────────────────────┤
│                         │
│      页面内容(滚动区)    │  ← scroll-view / page 滚动
│                         │
├─────────────────────────┤
│  [底部操作栏 / TabBar]   │  ← 贴底，避让 Home Indicator
└─────────────────────────┘
```

### 3.2 导航栏（小程序约束）

- **胶囊避让**：右上「••• ○」胶囊不可覆盖。自定义导航栏（`navigationStyle: custom`）时，用 `wx.getMenuButtonBoundingClientRect()` 计算标题区与胶囊安全距离。
- 标题居中、单行截断；返回箭头居左（二级页）。
- 导航栏随页面层级：Tab 根页**无返回**；`navigateTo` 二级页**有返回**。

### 3.3 TabBar（底部主导航，最多 5 个）

| 顺序 | Tab | 图标 | 说明 |
|---|---|---|---|
| 1 | 首页 | `home` | 概览 + 核心服务入口 |
| 2 | 服务 | `apps` | 服务目录 / 报价 |
| 3 | 订单 | `clipboard-list` | 订单 / 账单跟踪 |
| 4 | 消息 | `bell` | 通知中心（订阅消息 / 站内信）|
| 5 | 我的 | `user` | 公司 / KYC / 设置 |

- 选中态用 `accent/primary`（陶土橘）；未选 `text/tertiary`。
- TabBar 仅在 Tab 根页显示；二级页（详情 / 表单 / 支付）隐藏 TabBar，露出底部操作栏。

### 3.4 页面层级与转场

| 跳转 | 用途 | 转场 |
|---|---|---|
| `switchTab` | Tab 间切换 | 无动画 / 淡入 |
| `navigateTo` | 进入二级页（详情 / 表单 / 支付） | 右进左出 250–300ms `ease-out` |
| `redirectTo` | 流程内替换（支付成功后） | 淡入 |
| `navigateBack` | 返回 | 左缘右滑 / 返回箭头 |

### 3.5 安全区域

- 底部固定操作栏 / TabBar 须 `padding-bottom: var(--safe-bottom)`。
- 全屏启动页（渠道入口）须避让顶部状态栏与底部 Home Indicator。

## 4. 核心组件

### 4.1 按钮

| 级别 | 样式 | 用途 |
|------|------|------|
| Primary | `accent/primary` 实底 / 白字 | 页面唯一主操作（立即办理 / 提交订单 / 去支付）|
| Secondary | 描边 `border/divider` / `text/primary` | 次操作（上一步 / 更换）|
| Ghost | 纯文字 `text/secondary` | 弱操作 / 取消 |
| Destructive | `state/error` 文字或描边 | 取消订单 / 删除（二次确认）|

- 最小高度 **88rpx**；圆角 16rpx；按压态 `opacity:.85` + 150ms。
- **底部固定操作栏**：主操作居右且宽，左侧可放价格 / 次操作；贴底避让安全区。

### 4.2 单元格 Cell（WeUI 风格）

```
[图标块]  标题                      值 / 状态  ›
          副标题(Caption)
```
- 左图标块 18rpx 圆角单色底；右箭头表可进入；分隔线 1px `border/divider`，不通栏（左缩进对齐文字）。
- 点击整行高亮 `bg/sunken`。

### 4.3 服务报价卡（电商核心）

```
┌────────────────────────────────┐
│ [图标] 标准注册套餐      [推荐]   │  ← 推荐角标用 accent
│ 含 4 项服务                      │
│ HKD 5,800  ⟍6,800              │  ← 渠道价(强调) + 划线原价
└────────────────────────────────┘
```
- 价格层级：**渠道价**用 `state/error`/`accent` 700 字重大号；**原价**划线 `text/tertiary`。
- 推荐卡描边用 `accent/primary`；普通卡描边 `border/divider`。
- 整卡可点 → 服务详情；按压微升 2pt + 轻阴影。

### 4.4 标签 Tag / 状态徽标

- 圆角 12rpx，Caption 字号，**弱底色 + 状态色文字**。
- 订单状态：待支付(warning) / 服务中(info) / 已完成(success) / 已取消(neutral)。
- 账单状态（服务行内小标签）：待支付 / 待确认(info) / 已到账(success) / 已含(success) / 已作废(neutral)。
- 必须**文案 + 颜色双编码**，不可仅靠颜色区分。

### 4.5 步骤条 Stepper（KYC / 注册 / 流程）

```
①──②──③──④──⑤
合规 身份 地址 业务 提交
```
- 已完成 / 当前 = `accent/primary`（当前带光环）；未达 = `border/divider`。
- 顶部常驻，标明「第 N / 共 M 步」，可返回上一步。

### 4.6 表单与输入

- 字段：标签(Caption) 在上，输入框在下；圆角 16rpx，聚焦边框 `accent/primary` + 3rpx 柔光。
- 必填项标 `*`（`state/error`）。
- 校验：失焦即时校验，错误**行内**红字 + 图标；不弹窗打断。
- 证件 / 回单上传：虚线框上传区，上传后显示缩略图 chip（可删），OCR 结果回填并标注「待人工复核」。

### 4.7 价格 / 金额展示

- 金额 Mono 或 700 字重；币种 `HKD` 前缀小一号。
- 价格明细块：原价 / 渠道优惠（`state/success` 负号）/ 应付合计（加粗大号）。
- 收款账户信息用 `bg/surface` 块 + Mono 字体；转账附言（账单号）`state/error` 高亮提醒。

### 4.8 弹层 / 浮层

- 底部抽屉（Action Sheet）承载多选项；中部 Dialog 承载二次确认。
- 浮层阴影 `y=8 blur=24 rgba(0,0,0,.12)`，遮罩 `rgba(0,0,0,.5)`。
- 危险操作（取消订单 / 删除）→ Dialog 二次确认，主操作用 `Destructive`。

### 4.9 Toast / 加载

- Toast：居中或居底，2–3s 自动消失，非阻断；动词开头（「已提交」「已复制」）。
- 列表 / 详情加载：**骨架屏**，不用空白转圈。
- 长任务（AI 解析 / iCRIS 查册）：文字状态行 + 进度感（「正在解析对账单…」「智能查册中…」）。

## 5. 交互模式

### 5.1 手势

| 手势 | 行为 |
|------|------|
| 上下滑 | 浏览内容 |
| 左缘右滑 | 返回（二级页）|
| 列表项左滑 | 快捷操作（消息：标记已读；草稿：删除，红色二次确认）|
| 下拉 | 列表页刷新；详情 / 表单页不触发刷新，避免误操作 |
| 触底 | 分页加载更多 |

### 5.2 转场与动效

- 微交互 150–200ms，页面转场 250–300ms；进 `ease-out`、出 `ease-in`。
- 新卡片 / 新订单出现：淡入 + 上移 16rpx。
- 服从系统「减弱动态效果」：开启时降级为纯淡入淡出。

### 5.3 等待与加载

| 状态 | 表现 |
|------|------|
| 短等待 | 按钮内 loading 圈，禁用重复点击 |
| 列表加载 | 骨架屏（≤300ms 出现）|
| 长任务 | 文字状态行 + 步骤进度（OCR / 查册 / 生成）|
| 提交中 | 主按钮转 loading，防重复提交 |

## 6. 关键业务场景流程

### 6.1 新用户首启 · 整页渠道入口
全屏品牌页（深色 + 陶土橘光晕）→ 输入渠道码（或跳过自然流量）→ 进入首页。**已注册用户跳过此页**（本地标记）。

### 6.2 服务购买（电商主线）
服务目录 → 服务详情（报价 / 包含 / 流程 / FAQ）→ 立即办理 → 填必要信息 → 订单确认（价格明细 + 渠道优惠）→ 支付（线下转账 + 上传回单）→ 支付成功 → 订单页跟踪。

### 6.3 KYC 尽调（合规）
AML 合规说明（必须勾选）→ 证件上传 + OCR + 邮箱验证 → 地址证明 → 业务声明 → 提交审核。每步可返回，资料草稿自动保存。

### 6.4 核名
输入中英文名 → 前端实时格式 / 敏感词校验 → 智能预检 + iCRIS 查册（长任务状态行）→ 五级风险结果卡 → 确认锁定 30 天。

### 6.5 支付确认
账单（1 服务 ↔ 1 账单）→ 收款账户 + 附言 → 上传回单 → 状态「待确认」→ 运营核对到账 → 「已到账 / 服务中」。

## 7. 反馈与状态设计

| 场景 | 设计要求 |
|------|----------|
| 空状态 | 简短引导 + 示例 / 推荐入口，不留白板（如「暂无订单 · 去看看服务」）|
| 错误 | 行内提示 + 重试；网络错误说明可检查网络 |
| 离线 | 顶部条「当前离线」，提交类操作禁用并说明 |
| 授权 | **按需、即时**申请（要发通知时才 `requestSubscribeMessage`；要拍证件时才相机授权），附用途说明；拒绝后给手动入口 |
| 限流 / 失败 | 友好说明 + 稍后再试 / 联系客服入口 |
| Toast | 轻量、非阻断、2–3s、避让底部栏 |
| 危险操作 | 取消订单 / 删除草稿等二次确认 |

## 8. 无障碍

- 所有可点元素热区 **≥ 88rpx (44pt)**，提供 `aria-label` / 无障碍文本。
- 文字对比度 ≥ WCAG AA（正文 4.5:1）。
- 不以颜色为唯一信息载体（状态 = 图标 + 文案 + 颜色）。
- 支持系统字体大小；尊重「减弱动态」「增强对比」开关。
- 关键金额 / 状态对屏幕阅读器整段播报。

---

## 9. 文案与语气

- 简洁、口语、专业可信；移动端字少为贵。
- 状态文案动词开头：「正在查册…」「已提交，等待核对」。
- 错误文案：说清发生了什么 + 下一步怎么办，不甩锅、不吓人。
- 合规文案准确：KYC / AMLO / TCSP 等术语首次出现给一句白话解释。
- 占位符中性邀请性：「请输入公司英文名」。

## 10. 小程序特有考量

| 项 | 要求 |
|---|---|
| **rpx 适配** | 全量用 rpx；基准 750rpx；图片用 `mode` 自适应 |
| **胶囊避让** | 自定义导航避开右上胶囊，`getMenuButtonBoundingClientRect` 动态计算 |
| **安全区** | 底部栏 / TabBar `env(safe-area-inset-bottom)`；启动页避让上下 |
| **订阅消息** | 在「关键节点前」请求订阅（如年审提醒、KYC 结果、支付确认）；合规通知不可退订 |
| **授权时机** | 用到才申请（相机 / 相册 / 通知）；首启不批量索权 |
| **深色模式** | `app.json: darkmode` + `prefers-color-scheme` 全量适配；OLED 用 #1A1A1A 非纯黑 |
| **分包加载** | 后台 / KYC / 交付物等低频模块分包，主包控制在 2MB |
| **性能** | 长列表 `recycle-view` 虚拟化；`setData` 增量、节流；图片懒加载 |
| **中断恢复** | 表单草稿本地缓存（`wx.setStorage`）；切后台回来续填 |
| **登录** | `wx.login` 换 code → 后端绑定；`getUserProfile` 仅在必要时 |
