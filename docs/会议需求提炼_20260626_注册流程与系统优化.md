# Leapexbiz 会议需求提炼（2026-06-26）— 公司注册业务流程 & 系统优化

> **来源**：《公司注册业务流程及系统优化讨论 2026-06-26》会议纪要（说话人1=产品/技术，说话人2=瑞哥/业务）
> **范围**：对照香港公司注册处官方 **NNC1 表格 / iCRIS 实操**，逐页走查小程序注册主流程 + 后台框架定稿。
> **关键结论**：① 核名 = **近似度预检**（公司名 + 商标），不出价值判断、只给一条 AI 建议；② 注册表单要把**官方 NNC1 / 章程 / KYC 指引前置喂给 AI 重做**；③ 身份信息做成**公共模块**（一次填、多处带）；④ 新增**客户主数据档案**（公司 + 个人，多对多）——非常重要；⑤ 后台框架确认。

## 一、核名（近似度预检）重设计 〔P0〕

- **核名 = 两类近似度查询**，不做价值判断：
  1. **公司名称近似度**：返回与拟用名相近、仍在登记册的公司（昨天看到的反馈样式）。
  2. **商标近似度**：取名称**关键字**（如 STAR），在商标库查同字，返回各品类结果（会比较多，因为不知道客户要注册哪类）。
- **关键字识别用 AI**：全名查册时要能识别真正关键字（去掉"香港"等前缀/无意义字），稍微智能一点。
- **只给一条 AI 建议**：不做准驳判断，最多给"名称可用但商标可能不符"这类**合理建议**。
- **香港无"核名通知书"**（与内地不同）：预检通过 ≠ 最终能注册；文案要体现"仅供参考"。
- **命名与文案（做减法）**：
  - 对内地用户仍叫「**免费核名**」（叫"查册"用户看不懂；"免费查册"会让人犹豫）；下方小注"顾问人工"即可，**不要写一堆**。
  - **去掉自相矛盾文案**：现在又"AI 实时校验"又"智能预检"又"人工查册"，用户看懵。本质就是"**近似度预检**"（名称近似 + 商标近似），统一口径。
  - 文案要逐条抠（产品估计花 1–2 天）；没想好的先留空，简洁优先。
- **免费额度**：保留"**3 次免费**核名"（放在提交按钮下方），不能一直白薅。
- **价值**：把背后官方查询网站封装好，用户不用自己找；核名做成钩子 → 顺势推荐注册。

> 已落地的「人工核名中 · 1–2 天」结果页方向正确（本次确认）；下一步按"名称近似 + 商标近似 + 一条建议 + 3 次额度"细化，文案重写。

## 三、去 AI 幻觉 · 全局做减法 〔P0〕

- 删除 AI 凭空编造的内容：**"安心保证 / 安排会计师前可退 / 换牌之前"**等都是 AI 幻觉，去掉。
- 待办「**7 天有效**」去掉：订单**不设时效**、一直挂着，**可取消**（取消即作废）。
- 现阶段是**定框架/主流程**，不是抠细节；真正抠细节产品还需约 3 天。AI 不懂业务时自己生成会乱，要"做减法"。

## 四、注册主流程 + NNC1 表单（官方资料前置）〔P0〕

- **主流程确认**（昨日定稿）：付款后 → **核名 → 支付 → 提交注册信息 → KYC**（已改成此逻辑）。
- **本质 = 复刻香港政府线上"成立公司"流程**：登录注册处 → 成立公司 → 香港本地公司 → 填资料（公司信息/秘书/通知书/概要/签署/付款）→ 提交。把官方的 4–5 步压缩成小程序里"提交信息"几步。
- **关键做法：把官方文件前置喂给 AI 重做表单**（不要让 AI 凭网络资料臆造）：
  - **NNC1**（法团成立表格）空白版 + **批注样板（Sample A / Sample B）**。
  - **公司章程** Sample A / Sample B（香港有 A/B 版要求，AI 已识别，保留）。
  - **致商业登记署通知书**（IRBR1，代办类，客户不用填）。
  - 官方"成立公司"线上页面**截图**（3 张）直接给 AI 复刻。
- **现有表单要对齐官方、补字段**：
  - **中文名 / 英文名**：**二选一即可**（非两者都必填）。
  - **缺「经营范围 / 业务性质」列表** → 要补（官方有列表）。
  - **股本**：普通股 **Ordinary**、面值、总股本（保持简单，默认普通股即可）。
  - **注册地址**：默认用**法定秘书（我方）地址**；客户一般不用填。
  - **股东 / 董事 / 创办人（创办成员）**：每个人要带身份信息（中英文名、身份证/护照、通讯地址、**通常住址**、电邮等）——这些与 KYC 重叠（见 §五）。
- **下周拉香港实操顾问线上逐条过**：哪些字段**客户要填** vs **秘书公司代填**，按实操确认。

## 五、KYC 与「身份信息公共模块」〔P0〕

- **KYC 对象 = 董事 + 创办人（股东）**：当前 KYC 没区分；一期先定义"**董事 + 创办人信息必做**"，对象**取决于注册时填的人**（没填就不知道对谁做 KYC）。
  - 依据持牌时的 **AMLO Policy Statement / 官方 Guideline**（瑞哥手上有，会沉淀进文档）：客户尽调含**实益拥有人**。
  - **转入客户**（已有公司、不在平台注册）：也要做 KYC（只要提供服务就要尽调）。
- **一期 KYC 简化**：身份证 + **地址证明** + **业务证明**，别太复杂。
- **身份信息 = 公共模块（很重要）**：
  - 身份信息有两件事：**输入** 与 **核验**，可拆开；可把"**实名认证**"等步骤抽象前置（类似内地先实名再办事）。
  - **一次填写、多处带过来**：注册时填的董事/股东身份、通讯地址、通常住址 → KYC 不必再填一遍；用户**只填一遍**，按流程顺序前后自动带。
  - 身份信息是**多个环节共用**的公共模块（注册、KYC、其他业务都要），做成**可配置化公共组件**。

## 六、管理后台框架确认 〔P0〕

- **两个工作台 = 不同权限不同视图**：供应商工作台（只看自己的）、渠道商工作台。**渠道商保留**（818 就是要靠渠道推广，不只自营）。
- **客户管理**：见 §七（升级为客户主数据档案）。
- **订单管理**：下单 → 订单 → 分配 → 交付物。
- **商品管理**：
  - **不做组合商品**，所有服务**作为单项商品**（已落地）。
  - 商品可编辑信息；**详细描述用富文本**整段录入（已落地）。
  - **前置流程可配**（核名 / KYC / 提交资料等抽象为可配节点）：初期可**写死**（商品就 5–6 个），有共性的未来再做可配。
- **供应商管理**：新增供应商 → 配置**支持的服务** + **结算方式（固定 / 比例，按具体供应商配）**。
- **订单分配**：**按订单**分配给某供应商（一个客户多个订单可分给不同供应商，如商标给 A、财税给 B）；也支持"**该客户所有订单都给某供应商**"简化运营；**自动分配一期不做**（量小，做减法）。
- **订单结算**：订单**完成**后生成结算项；线下打款，系统标记"已结算"（运营/财务维护，真正打款/记账走别的 OA）。
- **渠道商管理**：新增渠道 + 基本信息 + **专属推荐码**；线索/成交统计（多少人填码、多少成交、成交额、佣金）；**每个服务可配不同提成比例**；佣金**按比例 / 按固定**。
- **佣金结算**：订单**完成**后生成一条佣金，按固定/比例，财务线下处理。

> 本次确认后台整体框架 OK，与昨日重构一致（已落地）。

## 七、供应商工作台 — 进度 + 交付物 〔P1〕

- 供应商登录看**分配给自己的订单** + 每个订单**当前进度**，可**更新进度**（"接单"概念去掉，太扯）。
- **上传交付物**（履约材料）：放在**订单层面**（不是供应商层面）。
  - 一个订单可有**多个交付**（如注册公司有多项交付物）。
  - 初期**不预先规定交付项**：供应商**自行新增交付**（标题 + 简单描述 + 一个文件 + 备注），想加几个加几个。
  - 量大后再做成**标准化/节点化**（如 CI 登记证、章程…固定项）。
  - 前台用户能看到订单**当前阶段 / 在做什么**（偏手动，后续结构化）。

## 八、客户主数据档案（公司 + 个人，多对多）〔P0 · 重要〕

> 瑞哥强调"**非常非常重要**"：交付/KYC/注册产生的核心信息要**回流沉淀**为客户主数据，用于续费提醒与长期维护。

- **两层档案**：
  - **公司档案**：公司名、CI、注册时间、注册地址、**周年申报日**、状态等。
  - **个人档案（人）**：小程序授权登录的人；身份证号、通常住址、手机、邮箱等身份信息。
- **关系 = 多对多**：一个人可关联多家公司，一家公司可对应多个人；可正查/反查，**数据结构要设计好**。
- **信息回流**：KYC / 注册填的身份与公司信息 → 沉淀到对应档案的**字段**（详情页要好好设计成"档案"）。
  - 做法：先让 AI 按官方服务流程/所需资料**梳理小程序要采集的字段**，再**反向落到后台**客户/公司档案字段，前后台衔接、信息齐全。
- **定期维护**：身份证、地址等**会过期/变更**，要定期维护。
- **驱动提醒**：**周年申报（NAR1）成立周年后 42 天**等续费/年审提醒，**基于档案**生成客户**待办**（交付核心信息 → 客户待办/提醒）。

## 九、防欺诈校验 〔P1，后置但必做〕

- 风险：小程序登录人与**董事/股东不一致** → 有人盗用他人身份证开公司（甚至洗钱/贩毒场景）。
- **要做一致性校验**（登录人 vs KYC/董事/股东身份），一期可后置，但务必规划。

## 十、RPA 自动注册（复刻政府流程）〔P1 / 二期〕

- 香港政府线上"成立公司"全流程可由会计在网站申请 → **可用 RPA 自动化复刻**（登录 → 成立公司 → 香港本地公司 → 逐步填资料 → 提交）。
- 注册、核名等"代理在政府网站操作"的环节，**后续都可 RPA 替代**（先人工，跑通后自动化）。

## 十一、优先级与节奏

- **优先级：EcoBox（EZBOX 会计系统）验收 > 小程序**。理由：先把会计系统定稿交给财务/顾问去验（产品可脱身 2–3 天），再集中做小程序，避免并行互卡。
- **EcoBox 现状**：能用 100%、但商用/可演示精细度约 60%；初始化还原度不足、银行流水解析已测（信息能提取）、票据解析待测；期末试算表支持 **Excel 导入**。
- **本周**：产品先把 EcoBox 弄到"可验收"状态 + 录操作视频；下周一去王帅处演示、清库后给客户测试链接。
- **小程序细节**：约**下周二（06-30）起 / 下下周上半段**；瑞哥下周拉**香港实操顾问**线上逐条过小程序流程（后台供应商部分也给看）。
- 产品抠注册/KYC 细节文案预计**再花 1–2 天**。

## 十三、香港合规 / 财务要点（校核）

- **核名**：香港公司注册处**不出核名通知书**，预检通过不代表最终核准（与内地不同）——文案须体现。
- **NNC1**：确有"**拟经营业务 / 业务性质（经营范围）**"字段，需补；中/英文名可只填其一；股份默认**普通股 Ordinary**。
- **注册地址**：默认用**法定秘书地址**（我方提供）。
- **周年申报 NAR1**：公司须于**成立周年日后 42 天内**提交 → 档案驱动 42 天提醒，准确。
- **KYC / 尽调**：依 **AMLO** 须查**实益拥有人（股东）+ 董事**；转入客户同样需尽调。
