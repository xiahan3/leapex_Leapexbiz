## 一、小程序需求

### R1 · 核心使用流程重排（P0，本次最大产出）

**四件事**：① 核名（钩子，可选、免费）② 提供注册资料 ③ 付款 ④ KYC。定稿顺序与规则：

1. **核名 = 免费钩子，做成独立"免费商品"**：用户进来可免费核名 → 输入公司名关键词 → 查香港公司注册处有没有近似公司 → 返回结果（有/无 + 近似公司名）。**结果保存为记录**，之后下单可复用。
2. **核名异步、不阻塞**：核名提交后台**人工预审**（预检/给建议，**不是 AI 终审，也不是我们审批通过与否**）；用户提交后**可立即进行下一步/购买**，不等审核。
3. **下单流程**：看服务 → 下单 → **先填部分注册资料**（先填信息再付款，转化率更高、显专业）→ 付款 → **KYC 放最后一步**（KYC 输入烦，前置会劝退）。
4. **已付款但没核名**：付款后生成**待办** = ①核名(若没做) ②填注册资料 ③KYC，按顺序走。
5. **每步要有「提交成功 + 引导」页**：如"核名约 1–2 小时/天，你可先做 KYC / 先购买服务，我们一起帮你"。
6. **核名 RPA 自动化**（一期争取上）：用 ICRIS 查册（账号登录 + 输名称 + 护照号 + 验证码 → 返回近似公司）。验证码简单、已调研可做；作为"免费核名"引流卖点。技术明天起预研。

> 流程图与两套方案：产品出 **2 个流程方案**给瑞哥选 + 几张流程图。

### R2 · 首页：新客 / 老客两版

- **新客版**：顶部介绍 + 「查看所有服务」+「找人工」（优先级提前，已做）+ 能帮你做什么 + 为什么选我们 +（基于画像）推荐服务。**新客保留推荐**。
- **老客版**（本次明确）：① **我的公司**（当前状态：注册中/正常…）② **待办事项**（如已付款但核名/KYC 未做就跳出来）③ ~~推荐服务~~ **去掉**。
  - 风格：**简洁、干净、丝滑**，参考**顺丰小程序**——进来就知道"我要干什么"，科技感 + 清晰，获客是次要。
  - 待办呈现：**进行中的订单**逐个展示走到**哪个阶段**（尽调/核名…）；下面挂**待办**（缺什么资料、在哪中断、提供什么才能继续）；点进去继续往下走。
  - 订单 = 有顺序的子任务流，要**定义好顺序规则**；未完成的步骤即待办。
  - 「待办」与「进度时间轴」二选一 → 选**简洁版**（订单 + 当前阶段），不要重复两块。

### R3 · 支付 / 收款

- **双币种**：港币 + 人民币。**不同时展示两币**（怕用户以为付两份），用**切换按钮**；**默认人民币**（初期国内注册多）。
- **收款主体**：先用**国内（上海）公司**收款，**微信/支付宝**（5,800 元无需银行转账）。香港版另做（阉割版/独立小程序/海外 TikTok）。
- 立百思收款账号（王帅发群）+ 确认能否**聚合码收款**。未来支持用户选付到香港 or 上海（两主体，可能两个小程序）。
- **退款**：先**不做退款功能**，人工线下退；但页面加**退款承诺话术**降低顾虑（如"安排会计师前随时退 / 会计师后 1 天内退 / KYC 不通过退款"）。

### R4 · 结构与可配置

- 底部 4 Tab：**服务 / 订单 / 消息 / 我的**（保持）。
- 首页广告/介绍/常见问题/服务内容 = **后台可配置模块**（需要就展示，富文本/长图），不写死。
- **办公租赁（生态合作）**页：富文本/长图；做成**留资入口**（留电话）→ 以**工单**转给生态合作伙伴 → 伙伴在后台跟进/反馈/结算闭环；可能先收**定金**（防跳单）。资料下周一去香港拿。

### R5 · 商品 / 服务（与后台「商品管理」对应）

- 价格/币种/描述/层级优先级**全部后台可配、重改**。
- **命名**：去掉"注册套餐"；注册=一个服务。商品分 **单项服务 + 组合服务**。
- **字段统一（关键）**：所有商品同一 schema——类型、单价、包含明细、服务内容、**适用人群/面向人群**、交付门槛、**前置流程要求**。不允许个别商品多挂字段。
- **适用人群**字段用于（新客/服务页）**画像推荐**，product 维护面向人群。
- **每个产品前置流程不同**（有的需核名、有的需 KYC、有的需用户前置上传资料）→ **影响开发**，需逐产品定制资料收集；瑞哥下周给清单。
- **核名规则校验**：真正注册时按瑞哥的「**核名指南**」文档做名称**前置校验**（香港禁忌词/需牌照词如 Trust/Bank 等）。

## 二、管理后台需求（重点 · 要重做）

### 后台模块层级（会上现场捋定）

| # | 模块 | 内容 |
|---|---|---|
| 1 | **驾驶舱** | 整体数据 |
| 2 | **客户管理** | 平台客户信息 + 客户公司信息 + **核名审核** + **KYC 审核** |
| 3 | **订单管理** | 购买订单 / 付款情况 / 交付物。**订单内不含供应商**（分配是独立动作）|
| 4 | **商品管理** | 卖什么：单项/组合服务、上架、**统一字段** |
| 5 | **供应商管理**（原命名错） | 供应商=乙方/履约方；入驻 → 填**结算规则**（固定金额 or 比例）→ 订单分配 → 维护进度 → 看结算 → 提现 |
| 6 | **渠道商管理**（被漏，需新增） | 渠道商=投流/推广方（前端 channel code 推荐人）；按**订单/服务**配**佣金（返点）**，不同渠道不同佣金 |
| 7 | **供应商工作台** | 看分给自己的订单 + 进度更新 + 结算金额 + 提现 |
| 8 | **渠道商工作台** | 看自己（投流带来）的订单 + 佣金 + 结算/提现 |
| — | 投流/线索 | **投流→二期**；**线索登记/跟进→一期**（企微/私信），初期可 Excel |

### 关键纠正（务必落实）

1. **渠道商 ≠ 供应商，拆两模块**：
   - **渠道商**：负责运营投流/推广（客户来源 channel code），收**佣金/返点**，可按服务/订单分别配比。
   - **供应商**：负责**履约**（乙方），按**固定金额/比例**结算服务费。
   - **生态合作伙伴**（如办公租赁，他收钱再分我）：本质也是"服务方/乙方"，订单分配给他、登录后台跟进；命名"供应商 or 生态"待定，但**都要有账号 + 工作台**。
2. **删/改定价矩阵**：价格 = 商品标准价（商品管理维护）；渠道差异在**渠道商佣金**，**不是每渠道一个售价**。
3. **商品字段统一**：单项/组合服务同一 schema（见 R5）。
4. **两个工作台**：供应商工作台 + 渠道商工作台；核心=**我的订单 / 进度维护 / 结算 / 提现**。结算方式：月结 / 每单结 / **用户确认收货后结**（类淘宝，7 天自动确认）——三选一可配。
5. **后台本质 = 商品/电商系统**：参考公司内**服务后台**（刘亮+佩奇）、**石角兽后台**（找荣亮要）、说话人2 旧"律师服务后台"。

## 五、风险 / 合规提示（产品+香港视角）

- **核名 RPA 用个人账号**：当前用瑞哥**个人 ICRIS 账号**做商业化 RPA，存在并发/反爬/封号/合规风险，需评估多账号或官方接口（一期可人工兜底）。
- **核名定位**：仅"查近似 + 给建议"，**非官方核准**；注册时仍需名称规则校验（禁忌/需牌照词）。**无 30 天锁定**（香港无此机制）。
- **KYC 必做**：合规刚需，可放付款后，但不可省；配"KYC 不通过退款"话术。**先填资料再付款**属体验优化，注意"先收款后审核"的退款承诺要写清，避免纠纷。
- **收款主体**：国内上海公司收款、香港履约，存在**跨境资金/合规**问题（说话人2 已提示），需法务评估；平台不宜沉淀资金，宜走**持牌聚合收款**（连连等）通道。
- **退款**：一期不做系统退款、人工线下退，但**页面承诺话术**要与实际线下流程一致。
