## 二、图标设计规范（依 Claude 设计规范）

Claude 规范要求：**现代几何 + 细线条（1-2px）+ 单色/双色 + 风格统一**。小程序全量采用 **Tabler Icons 线性图标**，禁止 emoji/彩色拟物图标（如截图中的 🗂️🔄👤 需替换）。

### 2.1 图标使用规约

| 场景 | 规则 |
|---|---|
| 线宽 | 统一 Tabler 1.5px stroke |
| 颜色 | 默认 `--text2`；强调用 `--accent`；语义色仅状态用 |
| 尺寸 | Tab 23px · 入口图标 20px · 列表 17px · 行内 12-14px |
| 容器 | 服务/分类图标置于圆角方形底色块（`border-radius:8px`），单色图标反白 |
| 一致性 | 同一语义全局同一图标（如"年审"恒为 `ti-calendar-check`）|

## 五、购买流程（完整电商体验）

### 5.1 流程总图

```
服务目录 → 服务详情 → [立即办理] → 信息填写 → 订单确认 → 支付 → 支付成功 → 订单页跟踪
            介绍/报价    必要信息表    价格明细+优惠   回单上传   生成订单     状态流转
```

### 5.2 分步说明

| 步骤 | 页面 | 内容 |
|---|---|---|
| 1 | 服务详情 | 介绍 + 报价 + 立即办理 |
| 2 | 信息填写 | 按服务类型采集**必要信息**（注册→公司名/股本/董事；年审→公司CI/BR；变更→公司+变更项）。新注册触发完整 KYC+核名+NNC1；存续服务触发轻量表单（必要时 KYC 复核） |
| 3 | 订单确认 | 服务清单 + 价格明细（原价 / 渠道优惠 / **应付合计**）+ 渠道码 + 须知勾选 |
| 4 | 支付 | 复用 V2.2 MVP：线下转账 + 收款账户 + 上传回单（账单↔服务 1:1） |
| 5 | 支付成功 | 成功页 + 订单号 + [查看订单] |
| 6 | 订单跟踪 | 进入订单页，展示服务进度与账单状态 |

### 5.3 必要信息表（按服务差异化）

| 服务 | 必填信息 |
|---|---|
| 新公司注册 | 走完整 KYC + 核名 + NNC1 五步（既有流程） |
| 年审 | 公司名称、CI 号、BR 号、财政年度结算日 |
| 董事变更 | 公司、离任/新任董事身份信息（新任触发 KYC） |
| 秘书变更 | 公司、是否转入 EasybookX |
| 地址变更 | 公司、新注册地址 / 选用 EasybookX 地址 |
| 股份转让 | 公司、转让方/受让方、股数、对价（系统预估印花税） |
| 名称变更 | 公司、新名称（触发核名流程） |
| 股份配发 | 公司、配发对象、配发股数 |
| 注销 | 公司、注销原因、税务结清确认 |

## 六、订单页（状态体系完善）

### 6.1 订单 / 账单 / 服务 状态机（去除不合理状态）

```
订单 Order：  待支付 → 服务中 → 已完成        （+ 已取消）
账单 Bill：   待支付 → 待确认 → 已到账         （+ 已驳回 / 已作废）
服务交付：    待启动 → 办理中 → 已交付
```

### 6.2 合并展示规则（订单卡）

| 订单状态 | 触发 | 颜色 |
|---|---|---|
| 待支付 | 已下单未付款（账单待支付/待确认） | 橙 |
| 服务中 | 已到账，交付办理中 | 蓝 |
| 已完成 | 全部服务已交付 | 绿 |
| 已取消 | 账单作废 / 用户取消 | 灰 |

> **去除的不合理状态**：原 V2.2 客户端混用了「待确认/已驳回」等运营侧状态置于订单卡顶层——现统一为客户视角的 4 个订单状态，账单/服务的细分状态下沉到服务行内的小标签展示，避免客户困惑。

### 6.3 订单卡结构

```
┌────────────────────────────────────────┐
│ 星辰科技有限公司            [服务中]     │
│ ORD-2026-0188 · 标准注册套餐 · 渠道LEX888 │
├────────────────────────────────────────┤
│ ⊞ 新公司注册   HKD 5,800   [账单·已到账] │  ← 服务行 + 账单状态标签
│ ⛉ 首年年审     HKD 0(含)    [已含]       │
│ ⊟ 公司印章     HKD 0(含)    [办理中]     │
├────────────────────────────────────────┤
│ 订单合计 HKD 5,800   [查看详情] [新增服务]│
└────────────────────────────────────────┘
```

### 6.4 订单分组
- **进行中**（待支付 + 服务中）
- **已完成**
- **已取消**（折叠）
