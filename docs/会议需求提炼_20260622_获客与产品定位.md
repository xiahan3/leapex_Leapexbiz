# Leapexbiz 公司秘书 — 会议需求提炼（2026-06-22）

> **来源**：《Hi Gym 产品运营及获客规划》会议纪要（瑞哥=业务/老板、产品运营专家、技术负责人）
> **范围**：仅提炼对 **Leapexbiz 公司秘书小程序 + 管理后台** 有价值的**产品需求**（剔除纯获客/IP/组织内容）
> **整理**：产品 + 香港财务理解
> **关键结论**：本次会议**重新明确了小程序的产品定位**，并暴露了现有"电商目录"方向与真实业务链路的偏差——需调整。

---

## 〇、定位重塑（最重要 · 影响全局 IA）

**结论：小程序 = 「顺丰式」交付物管理 / 履约平台，而不是获客/电商入口。**

- 老板原话：「第一步我们就很清晰，它就是个交付物的管理平台」「你打开顺丰的小程序就这个理解」。
- 业务链路真相（80% 场景）：营销/IP → **企微/线下顾问先建立信任、谈妥** → 顾问把小程序推给客户 → 客户在小程序内**自助走完当前阶段（如注册）+ 看进度 + 收交付物**。只有 ~10–20% 铁粉/渠道客户会直接自助下单。
- 老板自身被秘书公司服务过的**痛点**：单向发邮件、交付物用微信发、要找时翻文件夹、**看不到进度**、只能三天问一次"好了没"。

**对现有原型的影响（需调整）**

| 现状（电商化方向） | 会议要求（交付平台方向） |
|---|---|
| 首页/服务市场像"电商目录"，信息密度高 | 信息密度过高、新用户**不知道能帮他做什么**；要削弱目录、突出核心 |
| 「立即办理」直接进 KYC 上传身份证 | **信任缺失**：未经顾问沟通直接上身份证太突兀 |
| 强调"浏览报价→下单→支付"自助闭环 | 自助下单是少数场景；多数是顾问谈好后"客户来小程序走流程+看交付" |

> ⚠️ 不是推翻电商化，而是**降权**：把"交付物管理 + 进度可见 + 我的公司"提为一级；"服务市场/报价/下单"保留但作为已建立信任客户的自助通道。

## 一、需求清单（按主题）

### R1 · 首页区分「新用户 / 已注册用户」两套 〔P0〕
- **痛点**：当前首页信息密度过高，新用户进来不聚焦、不知道用什么功能。
- **需求**：
  - **新用户首页 = 咨询接待台**：突出 1–2 个核心入口（**新公司注册 / 公司年审**），其余服务收到二级；首屏先做"咨询接待"而非直接业务办理。新增清晰指引：「这个小程序能帮你做什么 / 大概花多少钱 / 多长时间 / 流程难不难 / 你需要准备什么」。
  - **已注册用户首页 = 我的公司 + 进度 + 交付物**：切换为"我的公司"卡片 + 二级详细入口（年审/变更/交付物/查询）。
- **落地**：复用现有 localStorage 区分逻辑；新用户首屏加一张"它能帮你做什么"的引导卡 + 核心服务大图标，弱化目录。

### R2 · 信任前置：弱化"裸上身份证" 〔P0〕
- **痛点**：新用户没跟顾问聊就被要求上传身份证 → 信任缺失、流失。
- **需求**：
  - 在进入 KYC/身份证上传前，**先给信任铺垫**：持牌 TCSP 说明、办理流程预览、"先与顾问沟通"入口（企微承接）。
  - 顾问推小程序时，小程序应能**直达客户当前阶段**（如"待提交注册资料"），减少客户困惑。
- **落地**：服务详情/立即办理前插一屏"流程与所需材料 + 联系顾问"；已有「问顾问/预约咨询」企微入口前移、加强。

### R3 · 企微 + AI 双层承接 〔P0 后台 / P1 小程序〕
- **痛点**：纯靠人承接忙不过来；90%+ 客服咨询是**售后**，售前靠人建信任。
- **需求**：
  - **企微群承接**：一个真人（关系型/转化）+ **群内 AI 机器人**（答常规/售后、辅助真人），一人可服务上百群（参考"开票员"群机器人模式）。
  - **小程序内 AI 客服/对话**：把常规咨询沉淀到小程序，减少人工重复答疑。
  - 后台：线索/工单 + 企微承接链路要先跑通，让投流流量"接得住"。
- **落地**：管理后台线索池已具雏形；新增"企微群-客户-订单"关联与 AI 答疑配置（MVP 可先人工+话术模板，AI 后置）。

### R4 · "准 AI" 服务流程（AI 逐步替代人工）〔P1〕
- **痛点**：香港雇人贵、不敢全职；要"客户感觉智能、背后人服务"。
- **需求**（渐进式）：
  - **核名 AI 辅助**：现背后是人；未来 AI 给名称匹配度/风险建议（人复核）。
  - **群内/小程序 AI 答疑**：常规问题 AI 兜底。
- **落地**：核名后台保留人工 iCRIS，前端加"AI 预检建议"占位；不改变合规人工终审。

### R5 · 业务品类做"全"：全生命周期 + 出海周边 〔P1〕
- **痛点**：客户要"什么业务都能在里面找到我"。
- **需求**：除注册/年审/变更外，逐步纳入：
  - **银行开户**（成立公司的一环，非独立业务）+ **跨境支付整合**（端口更便捷、内地客户付款）。
  - **HR / 香港强积金 MPF / 社保**类服务。
  - **记账报税**（与会计线 ECOBOX/云账房打通，结果回流小程序）。
  - 出海周边"引流型"小功能（需求面更广，做引流动作）。
- **⚠️ 香港财务修正**：
  - "香港社保"应表述为 **MPF 强积金**（雇主/雇员各 5%，月上限 HKD 1,500）+ 雇员补偿保险，无内地式社保。
  - **跨境支付**：平台不宜直接沉淀资金，宜接**持牌聚合收款**（连连等）做通道，避免无牌收款合规风险。
  - "开户"平台只能**协助/引流**，账户须客户本人与银行直接开立（KYC 由银行做），平台不可代客户开户。

### R6 · 资讯模块（视频号 ↔ 小程序联动）〔P1〕
- **痛点**：IP 内容与小程序割裂；视频号可直接挂小程序引流。
- **需求**：小程序内增「**香港出海资讯**」板块：整合政策解读/合规资讯（往期可查），与 IP 视频内容连接，沉淀关注者为客户。
- **落地**：新增资讯列表页（标题/来源/日期/正文）；后台资讯发布；视频号挂小程序跳转此板块或服务。

### R7 · 价格三档补"为什么选这档" 〔P1〕
- **痛点**：标准档标"推荐"但没说明为什么；用户不知道"花这个钱得到什么"。
- **需求**：每档套餐增加：**适用场景**、**包含明细**、**为什么推荐/为什么溢价**、**三档横向对比表**；高两档可引导"找顾问聊"。
- **⚠️ 注**：当前三档价（3,800/5,800/8,800）是 AI 随便给的占位，**待业务定价**后替换。
- **落地**：服务详情页已有"服务包含"，**新增**"适用人群/为什么选它/套餐对比"模块。

### R8 · 交付物中心强化：进度实时 + 历史沉淀 + 查询 〔P0〕
- **痛点**：交付零碎、看不到进度、要翻文件夹。
- **需求**：
  - 所有公司的**交付物 + 办理进度 + 节奏**实时可见（已有节点时间轴，需贯穿全部服务）。
  - **历史沉淀**：客户所有服务（注册/年审/各类变更）的交付物都归集于此，长期可查。
  - 增"查询"能力（按公司/服务/时间检索交付物与进度）。
- **落地**：现有"我的公司→交付物中心"扩展为跨服务聚合 + 检索；进度时间轴复用到年审/变更。

## 二、优先级与 818 发布范围建议

> 老板明确：**小程序 1.0 + 管理后台 1.0 必须 818 之前上线**，先跑通链路 + 几个种子用户，再大面积投流推广。

| 优先级 | 818 必做（跑通承接→交付链路） |
|---|---|
| P0 | 定位调整为交付平台（R0）· 新/老用户两套首页（R1）· 信任前置（R2）· 交付物中心强化（R8）· 后台承接链路（线索/企微）跑通（R3 后台侧） |
| P1（818 后迭代） | 群内/小程序 AI 答疑（R3/R4）· 资讯模块（R6）· 价格档说明与对比（R7）· 品类拓展开户/MPF/记账打通（R5） |

## 三、与现有产物的对照（要不要改）

| 现有 | 处置 |
|---|---|
| 首页电商目录感 | **改**：新用户首页弱化目录、突出核心入口 + 引导卡 |
| 立即办理→直接 KYC | **改**：插入信任铺垫 + 顾问入口 |
| 服务市场/报价/下单 | **保留降权**：作为已建立信任客户的自助通道 |
| 交付物中心 + 进度时间轴 | **强化**：跨服务聚合 + 查询 |
| 企微"问顾问"入口 | **前移/强化** + 后台线索承接 |
| 三档套餐 | **补**适用场景/对比/为什么推荐；价格待定 |

## 四、本文档不含（属获客/组织，另行规划）
视频号/抖音 IP 矩阵与人设、数字人口播流水线、投流 ROI、组织合并（运营部+出海服务）、818 倒排时间表、开发团队/进度——这些是**市场与组织议题**，不属本系统产品需求，故不纳入。但其中「视频号挂小程序」「企微承接」与产品有接口，已在 R3/R6 体现。
