## 一、核名查询：简繁转换 + 中英双名 〔P0〕✅

- **国内客户输简体、注册处以繁体为主** → 核名要**自动简→繁**再查；繁体框下方提示「已帮您转为繁体：XXX」。✅
- **中 / 英文名分别查**（大部分客户中英并存，但**互不需对应**：英文 `X Trading Limited` + 中文可叫「投资有限公司」均可）；可能出现**英文过、中文不过**，则提示改中文或不要中文名。✅（已拆「拟用公司全名（繁体中文）/（英文）」两输入）
- **查名结果不稳定**：顾问常查 2–3 次（同名不同次结果可能不同）；文案维持「暂时可用、**不保证 100%**」（注册处最终以繁体注册为准，近年也放宽部分简体）。
- 现「人工核名中 1–2 天 + 名称/商标结果页 + 3 次免费」方向不变。

## 二、身份证件与本人验证（KYC / QIC）〔P0〕✅

- **去掉「内地身份证」选项**，证件类型改 **香港身份证 / 护照 / 其他证件**。✅
  - 原因：**注册处认护照**（护照有本人签名可比对）；内地身份证只能写在 NNC1 格**外**、不进正式栏位；「一证两用」（既作身份又作地址）风险高。
  - **只有身份证、无护照的客户** → 谨慎，建议线下，且必须另取地址证明。
- **本人验证**：邮箱验证 → 改 **手机验证码 / 人脸识别**（二选一，验证真人 / 活体）。✅
- **防冒充（AMLO 防欺诈）**：NNC1 须**本人签字**，最好**面对面看着签**（防拿他人身份证冒名注册）；护照签名作比对。
- 资料项：证件 + 地址证明（水电煤近 3 个月）+ 业务性质（HSIC）+ 资金来源；**补充说明可删**。
- **PEP / 制裁筛查**：不强制、**界面先拿掉**（内部保留，需另购 KYC 数据服务）。✅（提交页已去 PEP 行）

## 三、章程与语言 〔P0〕✅

- **组织章程去掉「自订（英/中）」**，仅留**样本 A（推荐）/ 样本 B**；另加**章程语言：英文 / 繁体中文**。✅
  - 客户要**改章程条款** → **超出标准服务范围、不接**（一改牵动其他条款，需律师）。
- **语言规则**：全篇要么全英文、要么全繁体；**NNC1 与章程语言可不同**（NNC1 英文 + 章程中文 OK）；不懂英文的客户给中文章程；标准版约 17 页。
- **章程要交注册处但不用签字**（可选股东签）；**只有 NNC1 必须签**。

## 四、注册提交与交付物 〔P0〕

- **业务办理时效改「线下交 NNC1 后约 1 周取正本」**（顾问要跑两趟：一趟交表、一趟取 CI/BR 正本）。✅
- **防客户跑单**：**NNC1 预览 / 表格不给客户看到下载**（有客户拿去自己交后就甩掉秘书）；**章程可给客户看**；**注册类先收费**。
- **IRBR1（商业登记通知书）**：递交 NNC1 时须夹一张，选 **1 年 / 3 年 BR**（默认 1 年）；不夹会被注册处退回。
- **交付物 = Company Kit（整套约 6–10 本）**：CI + BR（正本已交注册处、客户拿副本）+ 章程 A&A + **绿色登记册 Register**（股东/董事/秘书）+ **股票本** + **原章（必须）+ 签名章** + **钢印（optional）**；**SCR 由秘书公司维护**（打印夹入空白本，非手写）。→ 后台交付物中心应涵盖这套清单。
- **物流**：线下寄原件，后台补**物流单号（快递单 + 地址）**，客户小程序可见；电子件后台上传即时可见。
- **签字**：香港**无电子签**，必须手动签；大陆客户快递寄表签字回寄。

## 五、支付 〔P1〕

- **聚合码（大陆支付宝跨境付款）尚未打通** → 暂两方案：① 香港账户先付、后台点掉确认；② 先付境内。客户**上传付款截图**。
- 现「双币（默认人民币）+ 港币线下转账（永隆）」框架不变；聚合收款持牌通道待接（永隆不行则换连连等）。

## 六、后台：渠道分销 + 供应商协同 + 地址供应商 〔P0〕

- **渠道分销**：818（8 月 18 日）大会推广；**1 万家代账公司**分销合作 → 客户进来输**渠道码**识别介绍人，**按订单设佣金率**，后台渠道商管理结算。
- **供应商协同**：超管把订单**分配给供应商**（会计事务所 / 秘书行）→ 供应商**登录后台**看订单 → **上传交付物 + 电子签 + 寄原件**；**小程序不开放给供应商**，供应商靠**通知提醒**（大陆微信、海外电邮为主；短期人工兜底）。
- **地址供应商页**：小程序增「**注册地址 / 办公室**」供应页（与达利国际·葵芳楼宇招商合作），放介绍引流。

## 七、目标客群与获客方向 〔背景〕

- **主客群 = 跨境电商出海公司**：国内电商合规新政（买单出口受限、税局按平台数据追缴）→ 大量转设香港公司（**亚马逊 1/3 新卖家为香港公司**），做「香港贸易公司归集 + 做实（地址/人员/决策）+ 税务居民 + 移民/CRS」架构。
- 特点：**收款多、交易笔数多**、用**平台账单 + 银行账户**做账（难一一匹配）、**主要求合规、对价格不敏感**。
- **获客**：产品做 **IP / 视频号**（出镜讲香港出海）、818 大会、香港团队信任素材（照片/内容）。

## 八、流程现实约束（产品需吸收）〔重要〕

- **注册最后一步仍需人工**：顾问**复核 + 线下交表 + 取正本**（约 1 周），无法全自动。
- 「线上提交」本质是**用客户 CR 账号代填**（灰色，服务机构本不能代填）→ 产品定位应是「**资料收集 + 交付/履约平台**」，不是全自动申报。
- **IRD eTAX / BTP 账号体系极复杂**：客户须**自己开 BTP 账号**，秘书才能被委任协助；秘书代开则要签字担责；**老年 / 不会电脑的客户很难自助** → 产品体验要尽量降门槛、多用人工兜底。
