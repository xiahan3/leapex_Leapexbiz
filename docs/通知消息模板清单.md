## 一、文档说明

### 1.1 用途

1. **研发**：作为 `message_template` 表的初始化种子数据与字段定义依据；触发事件即业务侧埋点/领域事件名，一事件一模板（或一事件多渠道模板）。
2. **运营**：作为消息中心「模板管理」的文案底稿，可在后台直接编辑标题/正文，无需发版。
3. **合规**：明确哪些消息必达、哪些可退订、哪些**绝对禁发**（见第五章）。

### 1.2 渠道口径（全平台统一，仅此 4 个）

后台模板编辑页的「下发渠道」为**多选勾选项**，选项必须且仅为以下 4 个全称：

| 渠道全称 | 说明 | 适用特征 |
|---|---|---|
| **小程序内部消息** | 小程序「消息中心」内的消息列表，可回看、可标记已读 | 全量消息的**默认兜底渠道**，所有模板必选 |
| **微信消息** | 微信小程序订阅消息 / 服务通知，需用户按次或长期订阅授权 | 强触达、时效性强的节点通知；受微信模板类目与字数限制 |
| **邮件** | 系统邮件（HTML），可承载附件与长文 | 交付物、对账单、报告、正式合规函件 |
| **短信** | 运营商短信（内地手机号为主） | 验证码、逾期告警、法定期限临近等**高优先级兜底** |

> 列表中为排版紧凑可简写为「小程序 · 微信 · 邮件 · 短信」的组合形式，但**下拉/勾选项在后台必须显示上述 4 个全称**。
> 旧文档中的「站内」「站内 · 微信」「微信服务通知」等写法一律替换为新口径。

### 1.4 发送原则

1. **合规类必达**：KYC/EDD、法定期限（年审 42 天）、政府文件签发等，**不可退订**，且必须至少覆盖「小程序内部消息 + 短信或微信消息」双通道。
2. **营销类可退订**：卡券、交叉推荐、活动公告，消息底部须带「退订/关闭此类提醒」入口，用户关闭后立即生效。
3. **避免夜间打扰**：短信与微信消息的下发窗口 **09:00–21:00（Asia/Hong_Kong）**；窗口外生成的消息进入队列，次日 09:00 统一发出。**例外**：验证码、安全告警（新设备登录）实时下发。
4. **同类合并去重**：
   - 同一客户 + 同一模板 + 同一业务对象，**24 小时内不重复下发**（幂等键 = 模板编号 + 业务对象 ID + 自然日）。
   - 同一客户名下多家公司触发同类到期提醒时，**合并为一条**，正文以列表罗列公司与到期日。
   - 同一订单短时间内多次节点推进，**合并为最新节点一条**（5 分钟聚合窗口）。
5. **渠道降级**：微信消息因未订阅/超出次数发送失败时，自动降级到短信（仅限合规类与到期提醒类）；小程序内部消息永远写入，不受降级影响。
6. **不打扰已完结对象**：订单已完结/公司已注销/卡券已核销后，相关营销与提醒模板自动停发。

## 二、消息分类

| 分类 | 代码前缀 | 定位 | 退订 | 默认渠道基线 |
|---|---|---|---|---|
| **合规通知** | `CP` | AMLO 尽调（KYC/EDD）、制裁筛查、合规资料留存与更新。持牌义务驱动，客户不可关闭 | 不可退订 | 小程序 + 微信 + 短信 |
| **流程通知** | `FL` | 从下单到交付的全链路节点：办理信息、进度推进、补交资料、核名/商标结果、交付物与快递 | 不可退订 | 小程序 + 微信 |
| **账单支付** | `PY` | 账单生成、在线支付、港币线下转账回单核验、催缴、收据与面议报价 | 不可退订 | 小程序 + 微信 + 邮件 |
| **到期提醒** | `DE` | 年审（成立周年日 + 42 天）、商标续期、地址/工位租赁、报税期限的递进提醒 | 不可退订 | 小程序 + 微信 + 短信 |
| **卡券营销** | `MK` | 免费核名券、立减券的发放/临期/核销，以及交叉推荐与活动公告 | **可退订** | 小程序 + 微信 |
| **渠道商** | `CH` | 渠道商申请审核、专属推荐码、名下客户与订单、月结佣金对账与打款 | 部分可退订（业绩简报可退订） | 小程序 + 微信 + 邮件 |
| **系统账户** | `SY` | 登录验证码、安全告警、账户与档案变更、系统公告、内部运营工单 | 安全类不可退订，公告类可退订 | 短信 / 小程序 |

## 四、年审到期提醒节奏

### 4.1 与法定 42 天的关系

香港私人公司须每年递交周年申报表 **NAR1**，法定期限为**成立周年日起 42 天内**，逾期由公司注册处按逾期时长**递增罚款**。因此提醒节奏以**法定截止日**（= 成立周年日 + 42 天）为基准锚点 `T`：

- `T` = 法定截止日 = 成立周年日 + 42 天
- `T-42` = 成立周年日当天（42 天法定期的起点）
- `T-60` = 成立周年日前 18 天（提前预热，让客户有时间准备材料与预算）

即：**第 1 轮属于「法定期前预热」，第 2 轮是「法定期开跑」，第 3–6 轮是「法定期内递进催办」，第 7 组是「逾期止损」。**

### 4.2 建议轮次表

| 轮次 | 触发点 | 相对成立周年日 | 建议渠道 | 文案要点 | 语气 |
|---|---|---|---|---|---|
| 第 1 轮 | `T-60` | 周年日前 18 天 | 小程序内部消息 · 微信消息 | 告知周年日与 42 天法定期，说明年审服务包含法定秘书 + 注册地址 + NAR1 递交，价格 HKD 4,422 | 平和提示 |
| 第 2 轮 | `T-42` | 周年日**当天** | 小程序内部消息 · 微信消息 | 宣告 42 天法定期正式开始，给出明确截止日 | 正式告知 |
| 第 3 轮 | `T-30` | 周年日 + 12 天 | 小程序内部消息 · 微信消息 | 强调剩余天数 + 一键下单入口 + 所需材料清单 | 提醒 + 行动引导 |
| 第 4 轮 | `T-14` | 周年日 + 28 天 | 小程序内部消息 · 微信消息 · **短信** | 首次点明「逾期政府罚款递增」，短信兜底 | 警示 |
| 第 5 轮 | `T-7` | 周年日 + 35 天 | 小程序内部消息 · 微信消息 · **短信** | 提供顾问联系方式与加急路径 | 紧急 |
| 第 6 轮 | `T-1` | 周年日 + 41 天 | 小程序内部消息 · 微信消息 · **短信** | 「明日截止」，最高优先级，不受合并去重压制 | 最高紧急 |
| 逾期 1 | `T+1` | 周年日 + 43 天 | 小程序内部消息 · 微信消息 · **短信** | 已逾期告知，说明罚款随时间递增、越早补交损失越小 | 止损 |
| 逾期 2 | `T+7` | — | 小程序内部消息 · 微信消息 · **短信** | 附逾期天数与补交路径，同步生成内部工单派给顾问 | 止损 |
| 逾期 3 | `T+30` | — | 小程序内部消息 · 微信消息 · 邮件 | 长期未处理的风险提示（可能影响公司良好状态），建议电话跟进 | 止损 |

### 4.3 执行规则

1. **每日 09:30（Asia/Hong_Kong）**跑批扫描，命中当日轮次的公司生成消息，进入下发队列。
2. **状态短路**：该公司当年年审订单一旦进入「已支付」及之后状态，**立即停发**后续所有轮次。
3. **多公司合并**：同一客户同一轮次命中 ≥ 2 家公司时，用 `DE-012` 合并模板替代多条单公司消息。
4. **轮次开关**：后台可按轮次独立配置渠道与启停，但 `T-1` 与逾期轮次**不建议关闭**。
5. **接收人**：客户（下单人）+ 公司档案中登记的公司联系人；同一人去重。
6. **新注册公司**：注册完成（`FL-014`）时即写入下次年审日，自动纳入次年提醒排程。

## 五、禁发与合规红线

| 编号 | 红线 | 具体要求 | 系统落地方式 |
|---|---|---|---|
| R-1 | **制裁命中 no tipping-off（最高优先级）** | 客户在制裁/名单筛查中命中并被冻结时，**严禁向客户发送任何形式的告知**，包括冻结通知、异常提示、进度停滞说明、审核不通过原因。客户端界面须表现为「审核中」的正常态，不得出现任何暗示性文案 | `CP-009` 接收人字段硬编码为内部角色白名单，客户身份不可被选中；风控状态为「制裁冻结」时，该客户的全部对外模板进入**静默拦截队列**，由合规官人工决定后续沟通 |
| R-2 | **不泄露他人隐私与完整证件号** | 消息中不得出现完整身份证/护照号、完整银行账号、他人手机号、他人住址。必须脱敏：证件号仅显示后 4 位；姓名仅显示姓 + `**`；账号仅显示后 4 位 | 渲染层统一脱敏函数；模板变量中禁止注册 `{证件号}`、`{银行账号}` 等原文变量 |
| R-3 | **营销类须可退订** | 卡券营销（MK）全部、渠道商业绩简报（CH-011）、系统公告（SY-009）须提供退订入口，退订后立即停发，且不得以任何方式绕过 | 模板表设 `可退订 = 是/否`；订阅偏好中心逐类开关；下发前校验订阅状态 |
| R-4 | **合规类不可退订，但不使用「强制」这一术语** | 合规通知、流程通知、账单支付、到期提醒、安全类系统消息为**服务必要通知**，客户不可关闭。**后台不得出现「强制」列或字段**，前后端文案统一用「不可退订」或「服务必要通知」 | 数据库字段命名为 `unsubscribable`（布尔）；后台列名显示「可退订」；订阅设置页对该类显示「服务必要，不可关闭」 |
| R-5 | **不作法律/税务结论式承诺** | 消息中不得出现「保证通过」「一定不会被罚」「零风险」等承诺；核名结论、审查意见等一律引用官方口径，并注明以政府机关最终决定为准 | 文案审核清单；模板发布前需运营 + 合规双人审批 |
| R-6 | **不夜间打扰** | 短信与微信消息下发窗口 09:00–21:00（Asia/Hong_Kong），窗口外入队次日发出；仅验证码与安全告警例外 | 下发网关统一时间窗校验 |
| R-7 | **群发须审批与人群圈选** | 任何群发/公告（MK-008、SY-009）须经人群圈选预览 + 二次确认 + 管理员审批后方可发出，并记录操作人 | 后台群发流程内置审批节点与操作审计 |
| R-8 | **合规留痕 7 年** | 所有已下发消息的内容快照、接收人、渠道、时间、状态须完整留存 **7 年**（AMLO 要求），并纳入审计哈希链，不可篡改、不可物理删除 | 消息记录表只允许软删除；写入审计链 |
| R-9 | **短信不承载敏感内容** | 收款账号、完整交付文件链接、KYC 驳回细节等敏感信息不得放入短信正文，短信仅作提醒并引导回小程序查看 | `PY-004` 等模板禁勾「短信」渠道；网关按模板做渠道白名单校验 |

## 六、状态与埋点建议

### 6.1 发送状态枚举

| 枚举值 | 含义 | 后续动作 |
|---|---|---|
| `PENDING` 待发送 | 已生成，在队列中（含夜间窗口等待） | 可取消 |
| `SENDING` 发送中 | 已提交至渠道网关，等待回执 | — |
| `SENT` 已发送 | 渠道网关返回成功 | — |
| `DELIVERED` 已送达 | 渠道回执确认送达（短信/邮件支持） | — |
| `READ` 已读 | 用户在小程序内打开该消息 | — |
| `FAILED` 失败 | 渠道返回失败或超时 | **支持一键重发**；自动重试 2 次（间隔 1 min / 5 min），仍失败则按 §1.4.5 降级 |
| `BLOCKED` 已拦截 | 因退订、订阅未授权、风控静默（R-1）而未发出 | 记录拦截原因，不重发 |
| `CANCELLED` 已取消 | 业务状态短路（如年审已下单）导致撤回 | — |

> 一条业务消息可对应多条渠道下发记录（1 条 message + N 条 delivery），状态分别记录；小程序内部消息的 `READ` 是「已读率」的唯一口径。

### 6.2 检索维度（后台「发送记录」列表筛选项）

- **接收人**：姓名 / 客户编号 / **手机号（支持模糊与精确检索）** / 渠道商编号
- **分类**：合规通知 / 流程通知 / 账单支付 / 到期提醒 / 卡券营销 / 渠道商 / 系统账户
- **渠道**：小程序内部消息 / 微信消息 / 邮件 / 短信（多选）
- **模板编号 / 触发事件**
- **状态**：上表 8 种枚举（多选）
- **业务对象**：订单号 / 公司名 / 申请编号
- **时间范围**：发送时间、已读时间
- 列表默认列：接收人 · 手机号（脱敏） · 分类 · 标题 · 渠道 · 状态 · 发送时间 · 操作（查看详情 / 重发）

### 6.3 指标建议

| 指标 | 计算口径 | 目标/用途 |
|---|---|---|
| 送达率 | `DELIVERED / (SENT + FAILED)` | 按渠道分别统计；短信/邮件 ≥ 98% |
| 已读率 | `READ / SENT`（小程序内部消息口径） | 评估文案与标题吸引力 |
| 失败率与失败原因 TOP5 | `FAILED / SENT` | 定位网关与模板类目问题 |
| 微信订阅授权率 | 已授权用户 / 活跃用户 | 低于阈值时提示引导订阅 |
| 拦截量与拦截原因分布 | `BLOCKED` 分组 | 监控退订流失与风控静默量 |
| 退订率 | 退订用户 / 曾接收该类用户（仅营销类） | 营销类 > 5% 需复盘文案与频次 |
| 提醒转化率 | 收到年审提醒后 T+14 内下单人数 / 收到人数 | 按轮次拆分，用于优化 §4.2 节奏 |
| 首响时长 | 需补交资料类消息发出 → 客户上传的中位时长 | 评估待办触达效率 |
| 平均触达渠道数 | 单条业务消息实际成功下发的渠道数 | 衡量降级机制有效性 |

### 6.4 埋点事件建议

`msg_generated`（生成）· `msg_sent`（提交网关）· `msg_delivered`（回执）· `msg_read`（小程序内部消息打开）· `msg_clicked`（点击 CTA 跳转）· `msg_failed`（失败，带 error_code）· `msg_blocked`（拦截，带 reason）· `msg_unsubscribed`（退订，带分类）· `msg_resend`（人工重发，带操作人）。

上述事件统一携带公共参数：`template_code`（模板编号）、`category`（分类）、`channel`（4 个渠道之一）、`biz_id`（业务对象 ID）、`user_id`、`ts`。
