# EasybookX 海外公司服务 — 商业策划与产品架构 v1.0

> **来源**：2026-06 客户研讨会议纪要（海外小程序事宜）+ 既有产品设计（TCSP 小程序 / 管理后台 / 审计报告）
> **定位**：把"会议里的极限思考"沉淀为一张可对齐、可执行的商业路线 + 产品架构
> **更新**：2026-06-11

---

## 一、一句话商业模式

> **用"持牌 + 一手资源 + IP 高地"建立认知高地，靠视频号自然吸引直客与渠道商，把香港公司全生命周期服务（注册→记账报税→税务→审计→地址→移民）做成套餐电商化交付，自营核心 + 供应商联营，按渠道×服务差异化分佣，数据回流小程序形成复购闭环。**

不养大销售团队，撬动自媒体 / 品牌高地获客。

---

## 二、差异化护城河（高地）

| 优势 | 说明 | 对手缺什么 |
|---|---|---|
| 香港持牌 TCSP | 合规资质，能合法提供公司秘书服务 | 多数国内代理无牌 |
| 自营香港运营团队 | 本地履约能力 | 纯中介无团队 |
| 上市公司一手地址 | 与 60 年老牌上市公司合作，拿一手房源注册地址 | 代理需层层转包 |
| 全链路自营履约 | 注册/秘书/合规自己做，不经多层代理 | 代理只能转单 |

> 战略含义：很多国内代理公司没有这些资源 → **它们会成为渠道合作伙伴**（放弃客户所有权换分佣）。杭州大型同行的先例验证了这条路通。

## 三、获客 GTM「高举高打」

```
个人IP + 公司IP 背书  →  视频号矩阵 / 直播切片  →  视频号 ↔ 小程序 打通
                                                      │
                          ┌───────────────────────────┴───────────┐
                          ▼                                       ▼
                   直客（自然吸引）                        渠道商推荐（分佣）
```

- 先把 IP 高地做起来：IP 本身代表产品（合规、香港财经专家身份）。
- 视频号讲产品 → 切片/工具沉淀在小程序 → 智能联动引流。
- 前期慢、后期越来越快；**通路必须先建好**。

## 四、承接转化（待完善 · 会议明确的缺口）

- B 端客户不能完全自助，需要**真人客服 + 企业微信**承接。
- **关键待定**：在用户旅程的哪个时点跳出"加企微"？客服是纯线上还是混合？
- 这是会议里点名"漏掉、要补"的设计点 —— 需从端到端用户旅程补齐。

## 五、变现 · 服务品类 + 分佣

**服务品类（套餐化）**
| 品类 | 交付方 |
|---|---|
| 工商注册 | 自营 |
| 记账报税 | 海外云账房 ECOBOX（代账公司供应商） |
| 税务申报（所得税 / 薪资税） | 自营 + 外包 |
| 审计报告 | 审计师外包（客户直签审计师，平台引流） |
| 注册地址 / 办公租赁 | 上市公司 ISV 联名（客户直签房东，平台是渠道/中介） |
| 移民 / 留学 | 后续拓展 |

**分佣模型**
- 按「**渠道 × 服务类型**」配置差异化佣金率 + 返利（如注册 15%、年审 8%）。
- 不同渠道可给用户不同价格（渠道专属价）。
- 结算线下，平台做记录与对账（见 V2.3 后台设计）。

## 六、履约模式（自营 + 联营）

| 模式 | 服务 | 签约关系 |
|---|---|---|
| 自营 | 注册 / 法定秘书 / KYC 合规 | 客户 ↔ EasybookX |
| 联营 · 审计 | 审计报告 | 客户 ↔ 审计师（平台撮合，规避法律责任） |
| 联营 · 地址 | 注册地址 / 租赁 | 客户 ↔ 房东（上市公司，需出水电煤/房产证）；平台=渠道 |
| 联营 · 记账 | 记账报税 | 供应商用 ECOBOX 做账 → 数据回平台后台 |

> 原则：涉及法律责任（审计签字、租赁出账单）的环节**客户直签供应商**，平台只做引流/撮合/分佣，不当二房东、不沉淀资金风险。

---

## 七、数据闭环

```
供应商履约（记账/报税/审计）→ 管理后台汇集 → 小程序展示报表 / 报税记录 → 续费复购
```
- 海外云账房 ECOBOX 的代账公司是平台的"服务供应商"；其做账结果回流后台，客户在小程序看到完整服务记录。
- 形成"工商 + 记账报税 + 税务 + 审计"的完整海外服务闭环。

---

## 八、支付方案（会议明确的必备项）

- 痛点：国内客户无海外账户，无法直接付款到香港账户。
- 方案：接**海外聚合收款**（如连连支付 LianLian）——客户用支付宝/微信 → 跨境到海外账号。
- MVP 过渡：银行转账 + 回单上传人工确认（已实现）；聚合支付为下一步。

## 九、1.0 短期目标（两步走）

| 步 | 负责 | 目标 |
|---|---|---|
| 1 · 运营 | 运营 + 业务方 | 小程序上线后的 0→1 运营方案：客户怎么来、种子用户怎么留 |
| 2 · 产品闭环 | 产品 + 研发 | 用户进来全链路打通：**先"能用"→ 好用 → 爱用** |

> 节奏：磨刀不误砍柴工——**先把"起点/终点/缺口"想清楚再开发**；小程序开发本身很快，关键在"想明白"。

## 十一、用户旅程（端到端 · 含承接时点）

| 阶段 | 用户行为 | 触点 | 平台承接 | 机会 / 缺口 |
|---|---|---|---|---|
| 1 吸引 | 刷到视频号/直播 | 视频号 IP / 切片 / 渠道商 | 内容种草 · 专家背书 · 植入渠道码 | IP 高地、切片沉淀小程序 |
| 2 承接进入 | 扫码进小程序 + 输渠道码 | 启动页渠道入口 · 企微码 | **企微 + 真人客服承接**（首次咨询/大额前） | ⚠ **承接时点待定（会议缺口）** |
| 3 选服务 | 看报价/详情 · 问客服 | 服务目录 / 详情 / 客服 | 报价透明 · AMLO 合规说明 · 客服答疑 | B 端非自助，客服转化是关键 |
| 4 下单支付 | 选套餐→填信息→付款 | 下单页 / 海外聚合收款 | 渠道价 · 聚合收款(连连) · 回单确认 | ⚠ **海外聚合支付待落地** |
| 5 履约交付 | 提交 KYC · 等待 · 查进度 | KYC/核名/注册 · 通知 | 自营+供应商履约 · 进度通知 | 交付物中心 · 快递追踪 |
| 6 复购留存 | 看报表 · 续费 · 推荐 | 我的公司/报表 · 年审提醒 | 记账报税回流 · 年审6轮提醒 · 转渠道商 | 一次客户→长期 · 客户变渠道商 |

> 情绪曲线：好奇 → 谨慎/犹豫 → 比较 → 担心/决策 → 期待 → 信任。两个"谷底"（承接、支付）正是会议点名要补的缺口。
