## 二、业务流程（后台已落图，见数据驾驶舱）

### 2.1 客户核心使用流程
```
填写推荐码 → 登录小程序 → KYC 核名 → 查看服务 → 购买服务 → 查看订单 → 订单完结
```
后台支撑：推荐码=渠道管理；KYC核名=客户管理；查看/购买=商品管理+订单管理；订单完结=交付物+订单管理。

### 2.2 渠道商入驻 / 履约流程
```
入驻 + 分佣规则 → 无推荐码订单分配 → 维护订单进度 → 查看佣金 → 结算提现
```
后台支撑：入驻审核+分佣规则矩阵=渠道商管理；派单=无码订单分配；进度/佣金/提现=渠道商工作台 + 佣金结算。

## 三、关键规则

### 3.1 无推荐码订单分配规则（模块 5）
自然流量 / 投流来的订单（无 Channel Code）由系统给出建议渠道、运营一键派单：
1. **优先级 1 — 地域/业务匹配**：内地客户 → 内地渠道商。
2. **优先级 2 — 负载均衡**：在办订单少者优先。
3. **优先级 3 — 金牌优先**（可选）：高分佣档优先派单。
4. **兜底**：无合适渠道 → 转直营运营承接。
> 分配后订单归属该渠道商，进入其工作台，并按对应分佣档计佣。

### 3.2 分佣 / 佣金结算（模块 5）
- **分佣档**：金牌 / 标准 / 基础，按「渠道档 × 服务」二维佣金率（渠道商管理页矩阵可配）。
- **计佣基数**：客户实付（渠道专属价，定价矩阵配）。
- **入账时点**：订单「已到账」→ 入应付佣金台账。
- **结算闭环**：渠道商发起提现 → 财务审核 → 线下打款 → 标记已结算。**客户不可见佣金**。

### 3.3 商品管理（模块 4）
- 商品 = 小程序「服务」目录数据源（套餐 / 单项 / 增值）。
- 维护：上下架、标准价、包含明细、适用人群、主要交付物。
- 前端服务市场与老客版「为你推荐」实时读取；渠道专属价在定价矩阵配置。

## 四、与真实后端的衔接（已接 `/api/tcsp/*`）

保留并复用既有桥接层，行为不变：
- 数据驾驶舱 KPI ← `/dashboard`
- KYC 审核列表/动作 ← `/kyc`、`/kyc/{id}/{action}`
- 账单回单确认 ← `/bills`、`/bills/{id}/confirm|reject`
- 佣金台账/结算 ← `/commissions`、`/commissions/{id}/settle`

> 新增页面（客户公司 / 商品 / 无码分配 / 投流 / 渠道商工作台）当前为静态原型，后端接口待按本结构补齐（建议新增 `/companies`、`/products`、`/orders/assign`、`/ads/stats`、`/partner/*`）。

## 五、RBAC 与渠道商端隔离

- **平台角色**：Super Admin / Admin / Operations / Compliance / Finance / Channel(只读) / Supplier(隔离)。
- **渠道商端（模块 8）**：Channel 角色登录后仅见**本渠道**数据（订单/佣金/提现），与平台后台同源不同视图，生产环境为独立登录 + 严格 `WHERE channel_id=current` 隔离。
